Audit Rules

The purpose of this functionality is to facilitate users for the creation & edit of audit rules. Audit rules are the configuration to log audit data for a system event for a particular duration. This helps the users at Head Office to audit the transactions going through the ETP system, especially in stores. Audit events are the transactions or actions the user does that could contain any suspicious behaviour.

The Audit Log captures the data as per the defined audit rules so that the stored data can again be retrieved as per requirement, in the form of audit reports.

The audit rule function will be available under the administration menu and be assigned to enterprise users. Users can create audit rules under a company, the records of the Audit Rule will be created at the Company level & the created rule is attached to the retail configuration template.

List Screen

  • On accessing the Audit Rules from the ‘Administration’ menu a list screen will be shown with existing audit rules created.

     
  • Users can also sort or use the search bar to search the existing audit rules by the Audit Rule Name or application on the list screen.  

Each record in the list screen has three dots 

’,  against it and they will have below following options

  • View - This option will open up the creation screen with pre-populated data only in view-only mode with all fields disabled.
  • Edit - This option will open up the creation screen with pre-populated data for editing, users can edit payment mode details.

Audit Rule creation

Users can create a new audit rule by clicking on the ‘

’  add button available on the main screen in the bottom right corner, a creation page will open up to create a new audit rule.

  • Audit Rule Name: The name of the audit rule to be created can be entered here. The user can enter any unique Audit Rule name for its identification (The rule name has to be unique only within the company).

     
  • Application: The application field will be used to select the ETP application for which the user wants to create and configure an audit rule. By default, the Storefront application will be selected in this field since the events will be captured related to the Storefront application. The audit events related to the StoreFront application will be available on the panel below.
  • Status: The status of the audit rule can be set to active or inactive while creating a new audit rule. Users can later on activate or deactivate existing audit rules by editing the audit.

Events Selection For Audit Rule

 

  • Event Group Panel
    • A panel containing the event groups of the Storefront application will be available on the left side of the section. The list of event groups will be given in the panel along with the event count of each event group.

       

Examples of Event groups for the StoreFront Application are as follows:

 

  • Customer Registration In Store Front
  • Billing
  • Sales Return
  • Order
  • Order Invoicing
  • Audit Event
    • Events are the individual user actions identified from an audit perspective to log its action data by all the users of the company performing those actions. On selecting an event group from the available event groups, all the available events will be available for the user where the user can configure the event parameters required for each event.

       

Examples of Events can be given as, For the Event Group - “Customer Registration in SF”  there will be the following events.

 

  • Discard during customer registration
  • Customer Edit
  • Walk-in Customer Data skipped
  • Walk-in Customer Detail Captured
  • Event Data Parameters
    • After clicking on the Event label or its toggle button, a section of multiple values with checkboxes will expand below it. The section will contain data parameters of the event. These are the event-related data that are recorded to view the event information in its audit report.

       
    • The parameters (event data) will be listed in the expanded area in the section with a check box for every parameter. Users can check the box to capture the parameter details in the event log.

       

Editing Existing Audit Rule

Audit Rules can be edited by clicking on the Edit option available in the 3 dots of the Audit Rule record on the list screen. When clicking on the Edit option, the edit window will open up where the users can make changes in the audit rule and click on the update button available in the top right corner to apply all the modified changes.

The users can modify the events and event data parameters by enabling or disabling the checkboxes under each event as required. Users can also modify the name of the audit rule and change deactivate or activate the audit rule as required.

Viewing Existing Audit Rule

Users can view the events and the event data parameters configured in the audit rule by clicking on the view option available in the 3 dots of the audit rule record on the list screen.

On clicking on the view button from the 3 dots of an existing audit rule record, the view page will open up displaying the audit rule name and the events and event data parameters configured but the user will not be able to modify any of the data in the audit rule.
