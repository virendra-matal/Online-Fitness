Use Cases for Company Master

  • The Company Master function allows authorized users to create a company, or multiple companies under a group company. The user can create and maintain separate products, inventory, and order information for different companies. 
  • A company is a legal entity formed by a group of individuals or stakeholders with the primary goal of conducting business activities, providing goods, services, or both, and generating profits. Companies are established to pursue various commercial objectives and often operate within a specific industry or sector.
  • The purpose of a company creation module in ETP Unify is to enable the setup and management of companies or businesses that you Use in ETP Unify to conduct their operations. This module available in  ETP Unify is designed to be used by multiple companies or retail chains, each with its own distinct settings, products, inventory, and reporting requirements.
  • Multi-Tenant Support: The module allows ETP Unify to support multiple companies or retail chains as separate tenants within the same software instance. Each company operates independently with its own data, ensuring data isolation and security.
  • Company Profile Setup: Users can create and set up the profiles of individual companies within ETP Unify. This involves defining company-specific information, such as business name, address, contact details, tax settings, and other relevant business information.
  • Sales and Transactions: The module tracks sales, transactions, and order history for each company separately. It allows for accurate reporting and analysis of company-specific sales performance.
  • Financial Reporting: Companies can generate financial reports specific to their operations, such as sales reports, profit and loss statements, and inventory valuation reports.
  • Data Segregation and Security: The module ensures data segregation between companies, protecting sensitive business information and preventing unauthorized access to data.
  • Consolidated Reporting: For administrators or higher-level users, the module may provide consolidated reporting that aggregates data from all companies for an overview of the entire ETP Unify.
  • In summary, a company creation module in ETP Unify facilitates the seamless onboarding and management of multiple companies or retail chains within ETP Unify. It provides the necessary tools and controls for each company to operate independently while benefiting from the shared functionalities and infrastructure.

Below are the steps to configure Company set up

 

List Screen

  • The company master which is available under Etp Unify → Administration → Company set up when opened  will show the card view of all the companies available under the selected group.
  • The card view screen will look like above screenshot. If no companies are available, then the screen will look blank.
  • The card will show below details,
    • Company Name and Status
    • Company Logo attached while configuring the company
    • Exchange Rate Type which is added while configuration of the company.
    • Currency which is added while configuration of the company.
    • 3 dot button which has a sub-option as edit, when clicked on edit it will open the create screen with all the configured company details already populated into the screen.
      • The User can edit all the details under the company except the group through the edit and update options.

Create Screen

Basic Information

  • The User can click on the button in the right bottom of the card view screen, to open the create screen.
  • The create screen will have the below details so that the User has to enter details to create a company set up.
    • Company Logo place holder where the User can upload the company Logo of the company being created, the same will be shown in the card view when the company is created. The image should be in 400*100 px size for the system to accept the image. The system will not accept if any image other than the said px has been uploaded and shows a error in the screen.
    • Company Code* is a unique mandatory free field where the User can enter the company code as a record in Unify for further Use.
    • Name* Here the User can enter the Company Name,this is a Mandatory column.
    • Group* The group the user has selected while logging into ETP Unify, in the session data will already be populated, the user cannot delete or edit this detail. This is a read only column.
    • Language* This is a drop down from where the user can select the Language used by the Company. This language will be made as the company Language from this selection onwards.This is a Mandatory field.
    • Establishment  Date* Here the user can select the establishment date of the company. This is a Mandatory field.
    • Default Node The user can select here the default node which can be set for the entire company. Once the same is set the node selected under this will be used as the default node on any company transaction.
    • Tax Declaration Number* This is a Tax declaration number where the user can save for further Use in Unify.
    • Reg Number* The User can enter the Registration Number of the company they are creating as a record in ETP Unify.
    • Company URL This is a free field to enter the Company URL of the specific company created. The URL should be a valid URL or the space will show an error display.

Company Address

  • Country* This is a Mandatory drop down field where the User can select the country of incorporation. Based on the selection, the city and state will be populated in the next columns as per the country.
  • State* This is a Mandatory drop down field where the User can select the State of the company. Based on the selection, the city will be populated in the next columns as per the State.This drop down will populate the City based on the Country selection in the previous drop down.
  • City*  This is a Mandatory drop down field where the User can select the city of the company. This drop down will populate the City based on the state selection in the previous drop down.
  • Addresses 1 and 2 These are 2 different free fields where the User can enter the address of the company being  created in below 2 address fields.
  • Postal Code*  This is a field to insert the Postal Code corresponding to the address entered above.

Currency Details

  • Local Currency*  This is a field to insert the currency that will be used by the Company for any transactions or exchanges. This Currency will be applicable to the stores/Nodes used created under this company.
  • Exchange Rate Type*The exchange Rate Type, which helps determine the local currency value against other foreign currencies.
  • Currency Rate Update Method* This has 2 options to mark by the User, The User can either select the Manual Rate Entry so that the User can enter the currency rate updates manually where its necessary or the system will automatically update in necessary places if Auto Rate Entry is selected.
  • Currency Rate Refresh Frequency* This will have 2 options to refresh the currency rate it can be either Hourly or Daily. The User can select either of those options.
  • Digit Seperator* the Digit Separator can be defined here as well. This separator will be used throughout the company for its currency display.

Other Information

  • Localization*  is captured from the predefined drop down values - this helps in case the company is from a specific country which requires specific localizations like BIR, GST etc. If your country has no specific localizations, the ‘Base’ option can be selected.
  • Product Date Manintainance at* - Group if this is set to group then the User will be able to see details like products etcc not only from the company he has logged in but also from the companies where the PDML(Product Date Manintainance Level) is Group. If its set to Company then the user can only view Products from the company only.
  • Time Zone* The appropriate Timezone can be set based on the company Localization or Incorporation.
  • Inventory Valuation*The inventory valuation can be specific as per options in the drop down by which the company follows.
  • Costing Method* This is to specify the costing method followed by the company. The User can Use the drop down to select the various types of costing Method.
  • Costing Level*  The User can select either Company level or Node level for the Costing set. If Cost is at Node level, in Product Master the costing method set in Replenishment details will be applicable for product related transactions.
  • Accounting year Period This is to mention the accounting year period the company complies to. The User can select the starting month of the Accounting time period from the drop down.
  • Date Format From this date format drop down, the User can select the date format that the company will follow throughout ETP unify wherever date is involved.
  • Date Separator can be defined here as well. This separator will be used throughout the company for its Date display.
  • Week Starting Day* Here the User can select the starting Day of the week, when the company starts to work.
  • After entering all the Mandatory columns above, the User can click on save to save the details and a new card view with the added details will be shown in the List screen after a success message.
  • If the User wants to discard the Screen will go back to the List screen and show the previous details already available.
