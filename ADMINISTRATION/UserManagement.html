Use Cases and Capabilities of User Group

A User Management Module in retail software is essential for managing user accounts, roles, permissions, and other aspects of user access and interaction within the system. Here are some use cases for the User Management Module in retail software:

  • Employee Onboarding and Offboarding:
    • Retailers can use the module to create new employee accounts, assign roles (cashier, manager, stock clerk, etc.), and grant appropriate permissions. When employees leave, their accounts can be deactivated or removed.
  • Access Control and Permissions:
    • Retail software can assign granular permissions to different user roles. For example, cashiers may have access to point-of-sale functions, while only managers can access sales reports and employee records.
  • User Authentication and Security:
    • Ensure secure login and authentication procedures to protect sensitive retail data. Implement password policies, two-factor authentication, and other security measures..
  • Role-Based Access Control (RBAC):
    • Implement RBAC to control access to specific parts of the retail software based on a user's role and responsibilities. This enhances security and compliance.
  • Multi-Store Management:
    • For retail chains with multiple stores, the module can manage user accounts, roles, and permissions across all locations while ensuring data segregation.
  • Compliance Management:
    • Ensure compliance with data protection regulations by managing user consent, data retention policies, and providing tools for users to exercise their data rights.
  • User Notifications:
    • Send automated notifications to users for account-related activities, order status updates, promotions, and other relevant information.
  • Customer Support Integration:
    • Integrate the module with customer support systems to streamline user issue resolution and provide a consistent user experience.

User Manual for User Management

The User can Navigate to ETP Unify→Administration→User Management to reach the module.

Card View Screen

When The User clicks on User Management the screen will open a Card view screen.

The Screen will showcase all the users available already if no users are available, then the screen will be blank.

The screen will have a search bar and the user can search based on User Name, User Group or Email ID.

The screen will also have a drop down to show values as Enterprise and Store. The Store Users cannot access this drop down since, they can only view and edit and Create Store Users. The drop will only show as store and it will not be changeable.

 The Enterprise Users can access this drop down and shift the drop down to view all the Store and Enterprises available for the companies he has access to in the List screen.

The screen has a store drop down to view the Users available in the List screen as per the stores they are assigned to.

There is a Status Drop down, where the User can view All statuses, Active,Inactive and Blocked Users available in the List.

The List Screen

The User can click on the blue plus Icon available in the right bottom of the screen, to open the Create Screen.

Basic Details

The below details will be available under Basic Details.

User Name*

Here the user can enter the User Name of the User for whom the specific record is being create.(Usually a name of the person).

User Type*

This is a drop down where the User can select the type of User they are going to create.

If the user who is creating the record is an Enterprise User then all the 3 options, Enterprise, Store,Vendor and Customer options will be shown. Customer and Store users can only use Customer,Vendor and Store Users.

User Group*

This is a drop down where the User Groups created based on the User type will be shown.The User can select any one of the values so that the rights applied to the User Group will be pre-selected in the below rights selection columns.

Email ID

Here the User can enter a valid email address of the User.

Mobile Number*

Here the Mobile Number of the user being created can be entered with the country code in the drop down.

Is Employee

This is a Radio button option where the User can mention if the User being created is an employee of the Companies/groups to be assigned below.If Yes then an input field will be shown where you can enter the Employee Code and this is a Mandatory column to insert. If Not an employee then the employee code column will not be available.

Language

This is a Drop down to select the Language of the User,When the Language is selected then that will be the assigned language of the user being created and this Language will be assigned as the default language for Login.

Email Alerts

When a User Checks on this box, the any email alerts that are sent from ETP Unify will be received to the above entered email address.

Over Ride User group access

When the User clicks on the check box the system will over ride the access assigned in the selected User group. This check box is not clickable when the “All Access” is clicked.

Active/Inactive Toggle

The User can use this toggle button to activate or deactivate  a user management record.

All Access

This is a Check box that can be clicked, when the User being created should have all the available Groups,Companies, Stores and Channels and Functionality. If this check box is checked then the Over ride User group access is not selected and is uneditable.

Only when the All access check box is unchecked then the below options will be 

available for selection

  • Add Company

The User can click on this button and the groups available under this login will be shown in the Left panel for selection as a multi selection option, when the User selects a group, the all the companies available under the group is shown below add company option.The User can Multi-select the companies that are needed to be given access for.

  • Add Channel and Node

When the User clicks on this button the Channels available under                          the companies previously selected will be shown with the company names on the left side bar. The User can multi select the Channels and Nodes needed under each company from the left side bar.

  • Add Functionality

When the User clicks on this button all the functionalities available with ETP Unify will be shown as Menus, Submenus and Functions. The User can multi-select the Menus,submenus and functions needed for the User being created.

Now the user can click on the save button to save the User Management record being created.

The record will not get saved if any Mandatory fields are not filed up or are incorrect.

When the Record is saved the User for whom the User management record will receive an email to the email address mentioned in the set record. The User can open the mail and set up a password as per the password policy set up in ETP Unify.

Password Set Up configuration

The above mail will be received, the User can click on the URL link to set up the password and when clicked the below screen will appear. The User can insert the new Password as per the password Policy available and can set the password by clicking on the “Set Password”. The same password can be used to Login further on.
