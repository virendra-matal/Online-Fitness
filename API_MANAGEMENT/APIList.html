The API List feature in Ordazzle helps the user publish, monitor and secure their APIs from the Ordazzle platform. 

There are two types of APIs users can create

Private: For this type of API, the user needs to add it in the client application in order to publish it. 

Public: For this type of API, the client application is not needed to publish it. Users simply need to provide some basic information in order to publish it.

Searching API Using Filters

The user can search for existing APIs using the search bar in the top section of the screen. They can search by either API Name, API Type or API Status. 

This screen would provide information related to all published APIs for the customer. These APIs can be in multiple statuses

Created - When the user provides basic details related to the API

Ready  - When the user provides an endpoint along with basic details

Modified -  After publishing the API, the user changes some information 

Published - When the API is ready to be consumed

Retired - When API is no longer needed

After finding the API, the user can edit or replace it by clicking on the ‘edit’ icon.

Add/Create New API:

By clicking the “+” icon in the bottom right corner of the screen, the user will be able to create a new API. In this screen, the user will have to input three pieces of information: API Name, API Type, Description. 

API Name:  Name of the api which user is creating

 API Type:  Private or Public API

Description: Description of the API which the user is creating 

Publish New API:

In order to publish any API, the user needs to provide some basic information.

API Endpoint: URL of the service/API which the user wants to publish. This is the only piece of mandatory information required for publishing.

 

API Definition:  Documentation link of the API

API Policy: This is an added security over the API. Currently a basic authentication policy is being provided which gives basic security based on username and password.

For the Private APIs, publishing does not mean it would be accessible for the consumers. In order to consume the Private API user would need to include the private API in at least a single Client App.

Edit - After clicking the edit option, the user can change the API Implementation, description of the API, and the API Definition. In the case of private APIs, users cannot change any details once the API is Published. 

Re-Publishing the API :

After making changes in the Public API, users can republish these changes by clicking on the “RE-PUBLISH” button on the top right corner of the screen.

Retire API


If the API is no longer needed then the user can RETIRE that API by clicking on the “RETIRE” button. This action will result in expiration of the API, and all the sources using this API will no longer be able to access it.
