The Client Apps Feature in Ordazzle helps users to give an added security layer to the private APIs which have already been created in the API LIST module. This feature provides additional security to APIs with API keys when a private API is added to a specific client app.

Consider following example : 

Scenario 1 -- Client App - Working as consumer and consuming service of other applications.

We can take an example of Ordazzle consuming services from Marketplaces like Lazada, Shopify etc.

We can create a Client App (public and Private) to consume those services.

The client App internally can be linked to an existing business process service to execute the business logic.

Scenario 2 -- Client App - Working as producer and exposing service for other applications to consume.

We expose our business process service via a Client App.

In both the cases above, Client App acts as an interface over the underlying business process service and hence shields the business service from getting exposed directly.

While doing that, we have two options of exposing this service.

a.    As a public service - It means that this service would be available to all clients of Ordazzle to execute.

b.    As a private service - It means that we can expose the service per client, This is safe and recommended. This adds an extra security layer by creating an API Key (encrypted complex value), which the consumer to the service must know and pass as part of the service request.

 

Searching API Using Filters

The user can search for existing Client Apps using the search bar in the top section of the screen. 

Add/Create New API:

By clicking the “+” icon in the bottom right corner of the screen, the user can create a new Client App. In this screen, the user will have to input two pieces of information: Client App Name, and Description.

Client App Name :  Name of the Client App which the user is creating

Description : Description of the Client App which the user is creating 

Register New Client App:

After providing basic information to create a Client App, the user will be redirected to this screen. Initially the Client App would be in the ‘Created’ status. Once the user provides required information to the Client App it can be registered. 

API Contracts: In this section all the private APIs which have been created by the user using API Creation are displayed. The user needs to select any private API from this dropdown to create a contract.

 

API Definition:  Documentation link of the API

API Policy: This is an added security over the API. Currently a basic authentication policy is being provided which provides basic security, based on username and password.

For the Private APIs, publishing does not mean it is accessible for all consumers. In order to consume the Private API user needs to include the private API in at least a single Client App.


The user can click on the “NEW CONTRACT” button to create a contract between a Client App and selected Private API.

After successful contract creation, the API appears in the table, with the endpoint details available in the API Details column.

This endpoint is only active if the Client app is registered successfully. To do so, users can click on the “REGISTER” button on the top right corner. Users can also add another layer of security by adding Client App Policy which can provide a basic authentication for the API using a simple username and password. 

Edit - Users can edit the Client App details or add any new Private API to the Client App by clicking on pencil icon or “CONSUME API” button.

Editing allows users to add new APIs i.e. to create new contracts with new APIs. Once these operations are done users need to RE-REGISTER the Client App using the “RE-REGISTER” button on the top right corner.
