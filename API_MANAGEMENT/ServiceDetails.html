The Service Details feature in Ordazzle helps users monitor services which are being used in the system. This feature is a great way to provide a clear picture of how service calls are performing and what could be the possible reason behind any low performance. 

Searching services using filters

The user can filter for any service using the filters in the top section of the screen. 

Reference ID

This is the reference ID generated for each service call in the application. This ID helps track any errors or issues which customers could face during the application lifecycle. 

Service Name

This is the name given to each microservice or restful service during the development phase to register the microservice on the EUREKA server. This is a unique identifier for each service. Users can see the performance of a specific service by selecting this service from the dropdown.

From Date -  To Date

This feature can be used either in combination or independently to check performance of a specific service or all services 

Response Code

Each service in Ordazzle returns a specific response code along with the response from the end service. These response codes would be different for each scenario.

The following would be possible response codes:

0000 - Success

1111 - Service side error

3001 - End service is down

3002 - Routing to end service failed

3003 - ActionId mapping not available

3004 - Connection refused to the end service

3006 - Internal server error

3005 - Unknown host exception

Refreshing/Clearing filters

Users can refresh or reset the filters by clicking on the refresh icon at the top right corner.
