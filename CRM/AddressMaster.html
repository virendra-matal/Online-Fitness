Address Master

An "Address Master" in CRM is a centralized database that manages all address-related information for customers or contacts. It standardizes, validates, and stores address data, ensuring accuracy and consistency. It integrates with other CRM modules, maintains address history, and enhances data security. This tool aids in improving communication, reducing errors, and supporting data analysis in customer relationship management. It helps organizations reduce mail delivery issues, improve customer satisfaction, and streamline business processes.

Objective:

Objective of this functionality is to facilitate multiple Address creation for customers as well as vendors. The functionality allows users to save & modify different types of addresses for a customer or a vendor that is helpful to simplify the customer & vendor creation process by having the address master as a separate functionality.

Address Master of Unify at ETP:

In the current version of Unify, there is an Address Master that is listed in the Menu bar of the CRM function on the left panel window. The customer or the vendor address at the Storefront is created in this Address Master.

The HO Users will mainly use this feature to create new or edit existing Addresses for different customers & vendors. The different types of addresses like billing address, shipping address and correspondence address  can be saved here and will be used for different types of transactions. The main screen shows these four columns i.e. Entity Type, Entity ID, Address Type and the Address.

Every row in the list will have three dots on the right panel When using the address master, users can click on the three dots of any record on the list screen. which on clicking will open an action bar which has View, Edit and Delete options.

Overall Functionality:

  • Any user who has the rights for Address Master Functionality can select the function in the menu bar to open the Address Master’s main window.
  • In the Address master main window, there will be a search bar on the top right corner of the screen where users can search the address of a customer or a vendor by Entity Type, Entity ID, Country, State & City of already existing addresses and that will be displayed in the list format. 
  • There will also be a Plus sign (+) at the bottom right corner on the screen which is used to create New Addresses for the Customers and Vendors.
  • In the New Address function, there will be 3 types of details that need to be entered. First is the Basic information that contains mandatory details like Entity Type, Entity ID, Address Type  and the Tax Registration Number (not mandatory).  Also, there is a box suggesting to make it as a default address if needed. Then there will be Location details to be filled that includes Address details, State, City, Country, Area, Pincode that are mandatory to save the Entity. Lastly, Communication details like Contact persons, Mobile numbers, E-mail id’s are to be filled but not kept mandatory.
  • The Entity Type can be selected as Customer or Vendor according to the requirements and Address types can be selected from billing address, shipping address and correspondence address.
  • In the case of “Set as Default Address” check box, only one address of an Entity ID can be saved as default address. Hence the user cannot check the check box for more than one address of an Entity ID. If a user checks the check box for an address when another address is already set as default, then the previously set address will not be used as the default address further.
  • In the top right corner of the screen are two options, one is the Save option to save the new or edited addresses and the second is the Discard option if it is not needed.
  • The addresses in the main screen will have options as Edit, View & Delete in their option list. The Edit will open the edit window which is the same as the creation window, but the Entity Type & Entity ID will be fixed. The other fields will display the previously saved data of the address which can be modified  & then saved. When clicked on View, the view pop-up for address will appear on the main screen. Delete will delete the record from UI as well as database.
