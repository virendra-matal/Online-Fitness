Age Group Definition

This functionality is to categorize the ages of customers as per companyâ€™s requirements. This functionality will capture the age range of customers and users can assign a name to that group and those created age groups will be shown to users during customer creation.

Age Group Definition List Screen:

o access Age group master:

  • Login in to Unify > Select CRM Module > Select Age group defination.
  • On click of Age group Definition,  below listview will be visible.

From Age: This will show the lowest age one must have to be in the defined age group.

To Age: This will show the highest age one must have to be in the defined age group.

Age Group Name: This will show the Name which is assigned to the selected age range by using this field. 

Add Age Group: The user will be able to open the age group creation screen to define the age group range.

 

Age Group List Record Option

The user will be able to see the list screen with all the defined age groups. It will be a single screen functionality.User can create new Age groups using the creation row. Just below the creation row there will be a grid showcasing existing age groups. Users can get access to the option list through the three dots. The option list will pop-up when clicked on the icon & the options will be Edit & Delete.

Edit: The user will be able to edit any records which are already created in the list screen. The Edit section will be opened in the same screen to edit the changes required.

Delete: The user will be able to Delete the already created age group as per preferences.

 

Age Group Creation Screen

From Age: The user will be able to input the Lowest age one must have to be in the defined age group.

To Age: The user will be able to input the Highest age one must have to be in the defined age group.

Age Group Name: The user will be able to input the Name which will be assigned to the created age range by using this field. 

Add: The user will be able to save the age group range defined from his/her end.

Discard: The user will be able to discard the changes.
