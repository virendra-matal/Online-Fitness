This enables the automation of the logistics function by providing integrations with selected carriers. 

There are 2 types of logistics setups:

Logistics Integration: The user can sign into different logistics partners which have been integrated with Ordazzle, allowing shipments to be tracked and delivery statuses to be updated automatically.

Non-Integrated Logistics Setup: The user can set up logistics which are not integrated with Ordazzle. This will require manually updating information and order statuses.

The user can get to the Carrier Integration screen by going to ‘Connections’ in the menu, and selecting ‘Carrier Integration’.

Carrier Channel Creation

Once the user reaches the Carrier Integration screen, they will see a list of countries on the left hand side. The user can click on one of these countries where he can either set up a new carrier account, or manage an existing one. In order to create a new carrier, the user needs to click on the ‘

Types of Carrier 

After the user has chosen to add a new carrier, they will be redirected to the carrier creation screen, which shows all available pre-integrated logistics providers for that country, along with a ‘custom’ option. Integrated Carrier

After the user drags a pre-integrated carrier to the drag & drop section at the bottom of the page, they will be able to click on the carrier and fill in the required information.

Name: The user can set a name for the carrier account. 

Node Serviceability: Defines all the nodes this particular carrier will be able to service. While assigning a carrier for a particular order, only carriers who have this node selected in Node Serviceability will be visible

Delivery Payment Method: This can be either ‘Prepaid’ or  ‘COD’. The user can also select both. This defines whether a carrier will be available to fulfill an order with this payment method. 

Product Carrier Service: This field displays the different service types available for the selected carrier. The service type will then be reflected while selecting this carrier for a shipment. 

Agreement Reference: This field can be used to record a reference number for a carrier agreement.

Agreement PDF Upload: The user can upload an agreement they have with the selected carrier.

Agreement Date: This field allows the user to capture the date of the agreement with the carrier.

Vendor Code: Any code assigned to the carrier.

Shipping Carrier Address: Contains the carrier's address for the user's reference. 

Manifest Mandatory: Yes/No - Defines if the user needs to get or generate a manifest in order to ship this order.

Return Reverse Logistics: Yes/No - Defines if the carrier will also handle logistics for return orders. This can include reverse airway bills.

Delivery Confirmation: Yes/No - Defines if a confirmation is required by the carrier to close the order.

After the user has filled in the above information and clicked on the ‘Save’ button, they will be directed to a second form to fill.

The user needs to input their carrier portal credentials onto this page, including the Account Number, Account Password, and Site ID. These details are used to connect with the selected shipping carrier and to integrate the user’s carrier account with their Ordazzle account. After the user enters these details, along with other information they may choose to enter regarding the carrier's contact information, they can click the ‘Save’ button. This will trigger a pop up message informing them that the account has been set up.

Custom Carrier

After the user drags ‘Custom Carrier’ to the drag & drop section at the bottom of the page, they will be able to click on the carrier and fill in the required information.

Serviceable Postal code: This option allows the user to download templates and then re-upload the populated template to set all possible postal codes which are serviceable by the logistics partner.

The Other parameters like AWB, Vendor code and Agreement File Upload will be set by the user depending on whether the user decides to enter this information.

Here there is no connection built up with any external source, so once the fields are filled up, the user will save the page and the Custom carrier will be created.

My Carriers

After creating a carrier, the user can see all created carriers on the ‘My Carriers’ screen. The user can see both integrated and non-integrated carriers on this page. Basic information like Payment Mode, Manifest, Delivery Confirmation, Reverse Logistics and the Service Product will be displayed here. The user can also choose to edit, deactivate, or adjust job schedules of these carriers by clicking the three dot option.

Edit: The user can edit any fields for a carrier, with the exception of the Name field. 

Activate/Deactivate: The user can activate or deactivate a carrier account, which will determine whether the carrier can be seen and selected during order processing.

Job Schedule: The user can see background jobs running for the carrier in order for carrier integration to work successfully.
