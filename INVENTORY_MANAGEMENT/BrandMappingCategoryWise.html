Brand Store Mapping use Case and Capabilities

Brand Store mapping is a simple process of mapping the stores to specific Products.

The User can here map the said Products to specified stores with the help of this feature.

Brand store mapping helps businesses keep track of which stores carry specific products. This information is crucial for inventory management, ensuring that stores have the right stock levels to meet customer demand without overstocking.

Marketers can use brand store mapping to plan targeted advertising campaigns. They can focus their marketing efforts on areas with a high concentration of stores that carry their brand's products.

As of ETP- Unified the app enables not only mapping of stores to the products, but also the user can map store attributes to product attributes to perform mapping for large amount of stores and products with minimal physical work.

Etp Unify facilitates the mapping of products and store as a unanimous module under a company for all the stores available Under the same company.

Brand store Mapping User Manual

The module is available under ETP Unify→Product Management→Brand Store Attribute Mapping. The User can navigate to the above section and make use of the Module.

Attribute Mapping Screen

The User will land on the above screen when Brand Store Attribute mapping is clicked on.
The User will now view 2 drop downs to select such as Store Attribute and Product Attribute ready for selection.
When clicked on the Store Attribute drop down, the drop down will showcase all the store attributes available in ETP Unify, Namely such as Store Brand, Store Concept, Store Subtype, Store Grade, Store Category, Store Group, Store sub Group and Store Cluster.(For these attributes, values can be created in Common Master to be made visible for mapping in the next screen.
 


When Clicked on the Product Attribute drop down, the User will be able to view the product Attributes Namely, Brand, Product Segment, Product Category, Product Sub Category, Product Class, Product sub Class.


Now the User can select any store Attribute and a Product Attribute that they are going to map.
The Purpose of this activity is to restrict or to attach a specific Product or a Product Attribute value(Multiple products with the same Product Attribute value) to a specific store or a specific group of store with the same Store Attribute value.
Hence when a user selects attributes from store and Product here, it implies that the User is going to map the values of the attributes selected in these 2 drop downs.
Then after selection of Attributes in both the drop downs the User can now proceed to click on Apply.
Incase the User wants to clear the selections, the User can click on Clear to clear the selections.
When Apply is clicked the below screen will open and the selected Store and Product Attributes will be displayed in drop down like icons(User cannot change this here)
 


Below the User will be able to view the list of attribute values from common master listed under the header of the store attribute selected previously.
Then the next column is the Product Attribute , when clicked shows the list of the selected product attribute’s values.
The Next 2 columns will show the Mapped and Unmapped attributes of Product attribute values when store attribute is selected and vice versa when Product Attribute is selected.
The User cannot perform the function from the unmapped column, the mapping is only possible from the unmapped column.
If the User wants to map attributes now, the User can click on the Pencil Icon given near the values under the unmapped attribute.
When the User clicks on the pencil Icon the below middle window opens up with the previously selected product attribute values from common master as of below.
 


Here the User can select the Product Attribute values that are supposed to be mapped to the store attribute value in the row entry.
The User can also perform the function with select All and Clear All button Options.
Once the Attribute Values are selected, the User can click on save and the same the count of the mapped Attribute values will be displayed under mapped Attributes.
The Remaining Attribute values that are not mapped will be shown in the unmapped column.
The above will be the same operation followed if the user maps from the Product Attribute row.(The Product Attribute will function will be the opposite dimension of mapping store Attribute and store Attribute values to the Product Attributes and Product Attribute Values.
Once the mapping is completed, The User can navigate out of the module by selecting other modules.
Now the User will be only able to view the Products mapped to the store (Mapping Performed in Brand Store Attribute Mapping) in Modules like Product Master - Node Details, Stock Transactions and Billing.
If the User wants to view the initial Store and Product Attribute mapping screen, the User has to unmap all the Attribute values mapped and then the screen will automatically return to the Landing screen.
The Above will be the steps and Uses to perform Brand Store Attribute mapping.
