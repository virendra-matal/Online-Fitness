Node to Node Relation - 

The objective of this module is to restrict and allow stock transfers between two Nodes or Stores. This allows the user to block or allow  Nodes from transferring Inventory within themselves and maintain a relationship for stock transfers. This Function is available under Inventory Management menu.

 

When the ETP Unify user Clicks on the Node to Node Relation, the first page the user sees is the List Screen. The user on this page is able to see all the Node to Node Relation that has been created with the Lead Time and the Status ( Status Color - Active for Green and Deactive for Grey ). 

Search Tab is available for the user to see the past record of the node in active and inactive Status.  The node name search criteria are for the Node in both the From and to relation.

Status - The status dropdown is available for the user to see the list of active and Inactive records.

Creating a new record for Node to Node Relation - The user has to click on the plus button in the list screen to create a new record. The user has to click on the Single Creation to create a single record. The user has to select from node from the Search/Dropdown and then select To Node from the Search/Dropdown. The user then has to enter the Ship lead time in Hrs as per how many hours it will take for shipment to receive from one Node to another Node. The user then has to activate the active status toggle button to make the relation active before saving.

As soon as the user saves the record, the record falls in the grid view of the list screen.

Create Bulk Record -  The user has to click on the plus button in the list screen to create a new record. The user has to click on the Download template then a CSV file is downloaded on the system. Once the template is downloaded the user needs to input all the mandate details like From Node ( Dropdown ) To Node ( Dropdown ) Ship Lead Time in Hrs ( Numeric Entry ) and Status ( Dropdown ).

 Once the user has entered multiple records then again clicking on the plus button the user has to click on the upload tab and upload the CSV file which the user has created. Once the Upload is successful the user will get a message that Node to Node Relation saved successfully.

   

Export - The user can export all the records of the node-to-node relation with all the statuses with Lead time. The CSV file will be downloaded to the System.

IF THERE IS NO RECORD CREATED FOR ANY NODE THEN THAT NODE IS ALLOWED TO CREATE TRANSFER ORDERS  AND RECEIVE TRANSFER ORDERS FROM ALL THE ACTIVE NODES OF THE COMPANY.
