This function allows the user to add and define various postal codes which can be serviced by different nodes for delivery. Postal codes will be defined based on Channels or Nodes.

Channel: This can be used while creating Order Receiving Rules, as postal code based rules can be set up to determine whether an order is put on hold or passed into the system

Node: This is used during Smart Order Routing in case the user wants to use ‘Serviceable Postal Codes’ as a criteria instead of ‘Distance’.

The user can add postal codes manually, or can upload them using Excel. They can also edit the existing postal codes and can update the From Date and To Date for postal code serviceability along with the Status.

Add Postal Code

Over here the user needs to click the ‘

’ icon to add a new postal code. After this they will be prompted to select whether they are creating a postal code for a channel or a node. Based on the selection, the user can select the respective Channel Shop or Node which they want to create a postal code for. Additionally, the user can select the country and the validity dates for the postal node. This is optional, and if left blank the postal code will be considered to not have an expiry date.

Upload Postal Codes

On this screen, the user can select a particular Channel, Node and validity dates, and download a template. After updating the template with the valid postal codes for that Channel and Node, they can upload the file, which will populate the screen with their postal codes.

Edit Postal Code

 

By clicking the edit icon, the user will be able to edit the selected postal code, as well as update the From and To Dates for the validity. The status can be updated here as well.

Search Postal Code

The user can use the search bar and search for postal codes linked to Channel Shops and Nodes, or alternatively search directly for the postal codes. Additionally, the user can also choose to filter postal codes by their status.
