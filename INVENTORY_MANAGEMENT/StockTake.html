Stock Take
 

Glimpse of Stock take Processing:

Unify Stock take caters to the problem of maintaining the big quantity stock in the retail stores.

Stock take supports to stock taking processing with both ‘Freezing Inventory’ and ‘Perpetual Inventory’

  • Freezing Inventory:While Doing the stock take, User wants to Put other stock related transactions ‘On Hold’ until the Stock take Completion, it is called ‘Freezing Inventory’. This type of stock assures no changes in the Stock quantity until the stock take is Completed.
  • Perpetual Inventory: While doing the Stock take, users don’t restrict the system to proceed other stock transactions (Stock-IN, Stock Out, Billing Etc); Then it is called as perpetual Inventory. In this type of stock take, System takes the screenshot of the stock quantity while stock take is initiated, and once the stock take is done it adjusts the stock to calculate the final inventory.

While Stock taking is in process, System allows your to:

  • Assign products from the same transaction to one or more account Persons and also maintain their Scanner ID.
  • View the current System stock and its Costing
  • Enter the stock manually
  • Runtime variance will be visible based on the system's stock and reported stock.
  • Update the system stock as per the latest entered (reported) quantity.
  • Do partial reporting and approval of the products.
  • Create the stock take for one or more locations within the same node at a time.
  • Do runtime monitoring on the Stock take processing.
  • Perform assigning and reporting via template (excel files) upload
  • Print the stock adjustment quantities for future references.
  • Update the Inventory as per the adjusted stock along with the Costing reflections
  • Settling the negative stock while doing the stock take. (Incase any product has -10 system stock and User reports 50 quantity, System auto-adjust the negative stock and final inventory will be 40 quantity.

Frequently Asked Questions:

Q. How many products can be selected while creating a Stock take.

A. There is No limit on how many products can be selected. Based on the linked brand with the node, User can select as many product as required.

Q. Is It possible to complete partial stocktake?

A. Yes, Users can assign, report and Approve/Reject the complete list of Products from stock take or partial once. In Partial processing User can select any no. products (not all) assign them, report them and then Approve/reject them. Users can repeat the same process multiple times until all the products from the stocktake are completed.

Q. Does Stocktaking have a File uploading process?

A. Yes, Users can ‘Assign the products’ and ‘Report the stock’ using File upload. Required templates for every stocktake transaction is available in the respective actionable pages. While User goes to the ‘Assign’ page of any particular stocktake, User can see the facility for ‘Downloading’ and ‘Uploading’ the templates. Same with the Reporting too. User need to make sure to have correct data in the same format and just by uploading the file the processing will be done.
