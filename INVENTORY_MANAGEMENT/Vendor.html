Vendor Master Document

Business Use cases of Vendor Master

  1. To List all the Vendors Available for a company so that the user can view and sort the vendors when needed.
  2. Serves as a Record as the Number of Vendors evolve in a huge quantity in terms of counts for any given company.

To sort vendors according to Products, Brands, Product subcategories, vendor groups,Vendor class and Industry segment so that the Business user can easily Filter the Vendor needs for the above filters with Quick and more efficient way.

  1. To save all the Details like address, products and Brands for each vendor and to diversify according to the same.
  2. Multiple addresses of the vendor can be stored so that the Order Re-allocation and other restocking activities can be planned accordingly.
  3. This Provision can store all the required information of the Vendor in terms of contact like Phone number and Email addresses and also the address to determine location accessibilities.
  4. The User can search with Parameters such as - Vendor class, Vendor  group ,Vendor  Code and Vendor  Name
     
  5. The Procedure to View and access Vendor master as of below.

1. When the Vendor Master in Inventory Management option in Unified is opened the List screen will be showcased.

2. This screen will contain all the previous Vendors already created, If no Records are created, then this screen will be blank.

3. The List will have the columns and details of - VENDOR CODE,VENDOR NAME,VENDOR CLASS,VENDOR GROUP, and STATUS

4. It also has a Active and Inactive drop down for the users to View the vendors accordingly and can also view both the statuses Vendors using all Drop down.

5. The three dot Button has 2 options - Edit and View

     (i) The edit option will allow the user to edit all the data except - Vendor Code. 

     (ii) The View Option is to showcase all the details available in any vendor and The screen will be non - editable.

  1. Creation of a Vendor

Kindly enter/Select the below details in the creation screen.

  1. Vendor Code  -  This is a field where user can enter any unique 15 Digit vendor code.(This is a Mandatory field)
  2. Vendor Name - This will be the free field where the User can enter the name of the Specific Vendor’s Name and alphanumerical characters can also be entered here.(This is a Mandatory field)
  3. Vendor Group  - In the vendor master the "Vendor group" will be a dropdown,which will consist of a vendor group which was created in the common master.(This is a Mandatory field)
  4. Vendor Product Brands- This will be a multiselect dropdown. Users can select multiple brands while creating a vendor. The selected brands will be mapped against the vendor so that user can identify which brand products are supplied  by the vendor. Its fetched from the product Master.
  5. Vendor Product Sub-Categories - This will be a multi select dropdown. The values in this field will be populated from the product sub category from the product master.
  6. Industry Segment - Industry segment identifies the market sector to which the vendor belongs to. For Example: Fashion, Electronics, etc.
  7. The Industry Segment will be defined under Common master and in the vendor master the "Industry segment" will be a dropdown, which will consist of Industry segment which were defined under Common master
  8. Valid From Date - The date from which the vendor will be valid for use in business.
  9. Currency - This dropdown field will by default show currency which is selected in the company master but users can change the currency if needed, because Vendors can also operate from other countries.(This is a Mandatory field)
  10. Vendor Class - In the vendor master the "Vendor class" will be a dropdown,which will consist of a vendor class which was created in the common master.(This is a Mandatory field)
  11. Is A Customer - This is a check box,If the user selects "IsACustomer" the Customer name field will be enabled.
  12. Customer Name - When a user clicks on IsACustomer then this Customer name field will be enabled, where the user can search customer name and the same will be attached to the vendor which is a customer.(This is a Mandatory field, if the above, Is A Customer is clicked)
  13. Mobile Number - Users can capture the vendor mobile number along with its Country code. (This is a Mandatory field)
  14. Telephone Number - Users can capture the vendor Telephone number along with The Country code (eg) +91 for India.
  15. Email-ID - Users can capture the vendor Telephone number along with The Country code (eg) +91 for India.(This is a Mandatory field)
  16. Tax Registration Number 1 - Vendor Tax registration number can be entered in this field.
  17. Tax Registration Number 2 - Vendor Tax registration number can be entered in this field.
  18. Tax Registration Number 3 - Vendor Tax registration number can be entered in this field.

4. Redirect to address Master.

(i)Once when the save button is clicked in the vendor screen, the screen shows a pop up if to redirect the same to the address to save the vendor’s address.

(ii) If Yes is selected in the above Pop up then the screen is re-directed into the Address master, then the user can add the address details and save the same.

(iii) If Discard is given in the “Would you like to add address for vendor code - XXX” The screen returns to the main Vendor list screen.

(iv) If the User wants to add address after returning to the vendor Master list screen, The user can anytime click on the address link in the same screen and get redirected into the address master , so that they can fill out the address details.
