The Sales Return function in Ordazzle allows users to create as well as process sales returns for any configured channel. Users can create their own sales return request here, or can also check and process returns initiated from the configured channels.

Sales Return screen elements 

The order processing screen also consists of 3 sections. 

Sales Return Tickers

The Sales Return Tickers are tabs available in the Sales Return screen to display the return orders in the logical groups based on status. The List consists of 9 tickers.

 Namely - All, New, Approved, InTransit, Received, QC Putaway, Sttl. Approved (Settlement Approved), Settled, Exceptions. 

The ‘ALL’ ticker displays orders of all the statuses. 

  • New - Displays return orders just created manually or received from channels.
  • Approved - Return orders with the status as Approved
  • InTransit - Return orders with the status as Shipped
  • Received - Return orders with the status as Stock Received
  • QC Putaway - Return orders with the status as QC Putaway
  • Sttl. Approved - Return orders with the status as Settlement Approved
  • Settled - Return orders with the status as Settled
  • Exceptions - Return orders with the status as Rejected
  • Error - Error orders that have products which are not available in Ordazzle are displayed here.

On clicking any ticker, the respective order list will populate.

Every ticker also has a count on it. This denotes the no. of orders available in the ticker. (for that company)

Order Actions 

This section consists of action buttons that are required to process a return order in Ordazzle. 

Users need to select the order that they need to process, then click on the desired action button.

Each button performs a different action on the return order. 

Bulk order processing & partial order processing is not allowed for return orders. Although, while creating or receiving return orders, partial lines of the order or order qty are allowed.

Not all action buttons will be available on the screen. Action buttons will appear as per status of selected order.

The detailed understanding of Return Order Processing is provided in its section. The list of order actions is as below -

  • Approve - Users can accept return intimation & start processing return
  • Reject - Users can refuse return intimation (in some cases, depending on the channel and their restrictions, the user may need to check this on the seller portal)
  • Arrange Shipment - Users can book return shipments using integrated carriers, or mark return shipments arranged by non integrated carriers. (Not required for MP logistics order)
  • To Receive - User can receive the returned stock to the appropriate inspection location
  • QC and Putaway - Users can perform quality check & add its data here
  • Approve Refund/Replacement/Exchange - After QC, users can process refunds, replacements or exchanges as per the return type of the order.
  • Payment - Users need to perform an extra step to add the payment details of the refund transaction using this action.

Sales Return List

Sales Return list is where you can find all the returns received from all Seller channels, as well as returns created by the user. The basic details of the orders are provided in the list.

The columns available in the Sales Return list are -

  • RETURN ORDER - This is the Return number received from the channel. This value is always unique for a seller channel. Users can also click on this number to open the Summary screen of the order.
  • RETURN DATE - This is the date-time at which Return request was placed on the channel or generated in Ordazzle. The value will be shown in the user's browser timezone.
  • ORDER# - This shows the order number of the original order against which return is generated.
  • NODE - This shows the Node name to which the order is allocated for Return receiving. The values will not be available for New orders.
  • CHANNEL/SHOP - This is the name of the seller channel defined in Ordazzle from where the order is received.
  • RETURN QTY - This shows the number of products available in the Return order. Each product can have multiple units returned.
  • STATUS - This displays the status of the return order. The different statuses are -

New - When Return intimation is received in the Ordazzle or created in Ordazzle.

Approved - When return intimation is accepted by the user will be processed further.

Rejected - When return intimation is rejected by the user.

Shipped - When the reverse logistics is arranged by the user for seller logistics return orders. 

Stock Received - When the Return order items are received at the selected warehouse. 

QC Putaway - When return stock is received & quality check is done on the returned items.

Settlement Approved - When the refund amount is confirmed for the return order with the type Refund.

Settled - When the return of refund is processed successfully based on the type of return.

Each order row also has a checkbox at the left side by which the order can be selected for processing.

At the right side of the row has the down arrow using which the user can expand the return order to see the expanded order view.

Order Expanded View 

When the user clicks on the down arrow in the return order list the order opens in an expanded view.

Order expanded view shows information -

  • Tags - User can apply any tag to search the return orders by tag. Applied tags can be seen & removed from here.
  • Delivery Date - It shows the delivery date of the original order.
  • Order Total - This shows the return amount of the order as received from the channel. In case the return is generated in the Ordazzle then this shows the sum of Line Net total of the original order that is returning here. Any header or line charges (exclusive) will not be considered in return.
  • Logistics By - This display if the return logistics is by Seller or MP. The value is only displayed if provided by Marketplace during return intimation.
  • Product Image - This shows the default image of the product from ordazzle. For variants if separate image/s are available the same will be displayed here.
  • Product Name - Name of the product as available in ordazzle is displayed here.
  • SKU Code - SKU code of the product as received in order is displayed here.
  • Unit - This displays the line quantity of the return order item.
  • Shipping Fee - If any shipping fee is charged for return then the same will be displayed here.
  • Return Amount (Line) - This displayed the Line Return amount for the order line.
  • Return Status Progression - This displays the progression of the Return order line status. All the past statuses of the order line are displayed here in the sequence. Along with status it also shows date & time at which the status was updated for the line.

Return Order Processing

Following are the actions that user can perform on Return Orders in Ordazzle -

Approve Return

By this user can accept return intimation & start processing return. Users can select any 1 return order with status New & click on the Approve button. This will open the following window.

The Approve window contains 

  • Return Order#
  • return product details that has : Return product code, name & image of the product from original order that buyer wants to return
  • Return Qty - Qty of the product that is requested for return
  • Reason - Reason selected by buyer for return. Seller can check this & decide for return processing
  • Return Type - This is the type by which the seller can compensate the buyer for the return. The options can be Refund (where seller can refund full or partial amount of the original item), Replacement (where seller can send new unit/s of the same product against return), Exchange (where seller can send different product against the return. Ex- Different color or size).The value can be preselected if coming from channel or it can be selected here if some unknown value (Other) coming from channel.
  • Delivery Date - This displays the date of original order delivery to verify against the return window limit, if any as per seller’s policy.
  • Order Total - This displays total or return order. Same value is applicable for refund for the Return & Refund type.
  • Logistics By - User can select if the reverse logistics will be arranged by seller or marketplace. Mostly if the original order is having logistics arranged by marketplace then the same applies for return as well.
  • Reason & Remark - Seller can select reason from predefined values & remark for his/her return acceptance.
  • Receiving Node - Use can select a fulfillment node where the seller can receive the return product. The system will show all fulfillment nodes of the channel to select from.

After entering the above data user click on save button. This will change the order status to Approved. If there is any marketplace call available then the system will also update the return acceptance at the marketplace.

Reject Return

Users can refuse return intimation requested by buyer (Note - make sure if marketplace allows to reject the return & update at marketplace as well else return may get auto accepted after some period). Users can select any 1 return order with status New & click on the Approve button. This will open the following window.

  • The Reject window contains all the fields almost similar to the Approve Return window, except it does not have the ‘Receiving Node’ field as return will never get shipped back if reducing the intimation. Also the reasons for the ‘Reject Return’ type will be displayed in this screen for seller’s selection.
  • After entering the above data user click on save button. This will change the order status to Rejected. Seller should also update reject return status at Seller channel incase API for Reject return not available. The respective message will be displayed in ordazzle’s popup as per channels.

Arrange Shipment

Now for orders having logistics by seller, users can book return shipment using integrated carriers or mark return shipment arranged by non integrated carrier us. (Not required for MP logistics order). Users can select any Approved order  by clicking on the ‘Arrange Shipment’ button.

  • Similar to the carrier selection in the order processing screen, users can select from integrated, non-integrated with manual AWB or List AWB carrier list. Carriers that provide return logistics will only be displayed in this list.
  • After selecting the integrated or non-integrated List AWB carrier user can add planned pickup date-time same it. The available tracking no will automatically populate for non-integrated with List AWB, while for integrated carriers it will get updated after successful shipment booking.
  • While for selecting non-integrated with manual AWB carrier users has to also enter the Tracking Number in the provided field before clicking on the save button.
  • After entering the above data user click on save button. This will change the order status to Shipped. For return with logistics by marketplace, this step is not required & should be skipped.

Receive Return Stock

Users can receive the return stock on an appropriate inspection location using this action. For orders with status shipped or Approved, users can select the order & click on ‘To Receive’ button. It will open the following window.

  • Under Inspection Location - User can select any one of the Inspection location (Location Type = Inspection) of the receiving node at which the returned product will be received & stocked.
  • Remark - Users can also add any remark here.
  • After entering the above data user click on save button. This will change the order status to Received.

QC and Putaway

Users can perform the quality check on the received return product & add relevant data using this action. Users can select any Received order & click on QC and Putaway button.The following window will open.

  • Product section - It will show product identification details as Code, Name & return Qty
  • QC Images & Video - User can upload max 5 images & 3 videos per returned product of the quality check performed on the returned stock.
  • QC Result - As per product’s condition, user can select as QC Pass (if product in good condition) or QC Fail (If product is not in good condition)
  • QC Reason - Based on QC Result selection relevant reason options will populate to select from.
  • Remark - User can also add any remark if required
  • Inventory location - User can select the location where the returned product will finally be stored. The QC Passed product can be stored in any of the approved locations so the same will be available to allocate to any new order. The QC Failed product can be stored in any rejected location so it will be separated from allocatable stock. Relevant locations based on selected QC Result will populate for selection
  • Batch/Lot/Serial No - If the returned product is lot controlled then its lot / serial no. can be entered here. If the reference of the original order is available then its original lot/serial no will be auto displayed here, else can be entered by the user.

 

Approve Refund/Replacement/Exchange

After QC users can process refund, replacement or exchange as per the selected return type of the order. Users can select any QC and Putaway order & click on Approve Refund or Approve Replacement/Exchange button.

  • If the Return Type is Refund, then the screen will display the return amount & Remark. Here users can confirm the return amount to refund to the buyer & can enter any remarks if required.
  • If the Return Type is Replacement then the user can select the replacement order in the field, this should be the regular order from the order processing module that is being processed & sent as replacement for the return. It will consist of the same seller channel &  product as the return order. Users can also add remarks here.
  • If the Return Type is Exchange then the user can select the exchange order in the field, this should be the regular order from the order processing module that is being processed & sent as replacement for the return. It will consist of the same seller channel but different products from the return order. Users can also add remarks here.
  • After selecting the required data & clicking on the Save button in above windows, order status will be saved to Settled for return type Replacement/Exchange and Settlement Approved for return type Refund.

Payment

Incase of refund, the user needs to perform an extra step to add the payment details of the refund transaction using this action. Users can click on any order in Settlement Approved status & click on Payment action. It will open the following window.

  • Return Amount - The return amount of the order will be displayed here for reference.
  • Payment Ref 1, Ref 2 & Ref 3 - These are the Payment Reference fields for the Refund transactions performed by the seller. Users can add transaction ID, document no., token no. or any other reference value of the transaction here.
  • Payment Date - Users can add the date at which the refund transaction was done.
  • Remarks can also be added in this action window.
  • On click of save the entered transaction details will be saved & order status will change to Settled.

Return Summary View

The summary view is the detailed view of the return order. This can be opened by clicking on the Return Order# in the row. The summary screen consists of the following data.

Return Order Summary

This section contains all the header level details of the return order. You can find the following details in this section.

  • Return Order Number
  • Order Total (Return Total)
  • Delivery Date
  • Customer Details - It has Customer’s Name, Phone Number, EmailID, Shipping Address & billing Address of the original order or as sent by channel.

Status wise details

This section shows the status wise details provided by the user or updated in the return order (Like Tracking No. received from Integrated carrier after arranging shipment).

  • The status wise details are displayed in the following sections.
  • Approve Return
  • Refund Return
  • Return Shipment
  • To Receive
  • QC and Putaway
  • Approve Refund
  • Approve Replacement/Exchange
  • Payment

Other Functions in Return List

Manual Return Creation

  • Users can create return intimation manually for the already delivered orders from Ordazzle. User to click on + button in the return list which will open the return creation screen.
  • It opens the order selection screen which will display all the previously delivered orders from ordazzle. Users can search here based on Order No, Product Code, Customer Name, Seller Channel.
  • Users can select any order from this list using the SELECT button. This will populate its data in the return creation screen.
  • Return creation screen will display the details of order as Order Total, Delivery Date, Customer Name, Delivery Address, Channel/Shop, Logistics By (of delivered order), Delivery Method (of delivered order).
  • Then it also has a product section that displays all the product lines available in delivered order. The product details as Name, Code, Image, Order Qty will be displayed here.
  • It also has a checkbox to select which of the order products the user wants to add in this return intimation. It can be further controlled at line quantity level using following fields
  • The quantity level fields available in the section are,
    • Returned Qty - If this order product has already returned & its return process is completed(settled) then its line Qty is displayed here.
    • Returned Intimation Qty - If this order product has already in return process but not settled then its line Qty is displayed here.
    • To Return Qty - This displays the final Qty that the user wants to add in this return intimation. This will by default show the value

= (Order Qty - Returned Qty - To Return Qty)

  • Users can reduce this To Return Qty or adjust back up to the calculated point using the + and - buttons.
  • After selecting the orders with desired products for return & return qty, user should also select the Return reason as communicated by buyer also with remark if any.
  • After adding all data & clicking on save this will created as a new return order & appear is list with status as New.

Order Filter

  • Users can filter the list of return orders using the Filter icon available on screen. On click of the button it opens the filter panel that has different criteria to apply order filters.
  • Users can select one or more filters at a time & apply it. Applied filters can be seen in the order list & users can remove any criteria from there as well.
  • Users can also save a particular set of filter criteria using the ‘Save Filter’ option. This will create a filter view that can be used for personal view or can also be shared with others using the button of ‘Share with others’.

Return Order Export

  • Users can export selected orders in the pdf document to get all the order data in tabular format.
  • If the user wants to export return orders of any ticker then the user can click on Export without selecting any order. All the orders of the current ticker will be considered for export.
  • Else user can also manually select multiple orders and export them

Order Search & Sort

  • Order can be searched in this field using different order fields like Order Number, Return No, Tracking No, Order Date, Order Tag, etc.
  • Users can also sort the orders based on criteria as latest to show first or oldest to show first
  • If the user wants to export return orders of any ticker then the user can click on Export without selecting any order. All the orders of the current ticker will be considered for export.
  • Else user can also manually select multiple orders and export them

Order Search & Sort

  • Order can be searched in this field using different order fields like Order Number, Return No, Tracking No, Order Date, Order Tag, etc.
  • Users can also sort the orders based on criteria as latest to show first or oldest to show first.

Refresh

  • Users can click on the refresh button provided on the screen to get the updated records of orders in the list screen, in case any user has processed any order in another instance or if order is getting updated by system.

Company selection

  • As like other masters users can change the company at the top side of the screen to see records of other companies. User can see all the companies that he/she has access to.

Marketplaces Return Order Processing in Ordazzle -

Return orders of following marketplaces can be processed from Ordazzle & it will update the changes in the marketplace.

Lazada

  • New return order placed on Lazada will also be received in the Ordazzle.
  • Users can approve the Return Order which will accept the Return on Lazada as well (with ‘Return and Refund’ option).
  • Users can also reject the Return (‘Reject Return’ option) Order if the user wants to reject the return on Lazada.  (Note - If the user wants to do ‘Refund without Return’ option then kindly select the option on Lazada without processing the order in ordazzle. As Ordazzle does not support the option.)
  • Also when receiving a return in Ordazzle it will also update the Return Received on Lazada. (Note - Hence Receive return should only be performed in ordazzle when the return parcel is actually received at seller).
  • When approving the refund in ordazzle it will also Approve the Refund on Lazada & initiate refund on Lazada.

Shopee

  • New return orders placed on Shopee will also be received in the Ordazzle.
  • Users can Receive the Return Order after accepting the return in Ordazzle which will accept the Return on Shopee as well. This will approve the return & initiate refund on Shopee.
  • If a user wants to Reject the return then the user can perform the action in Ordazzle but the return order must be manually rejected on the Shopee platform as well. As Ordazzle does not support Reject Return for Shopee.

Zalora

  • New return orders placed on Zalora will also be received in the Ordazzle.
  • Users can approve the Return Order which will accept the Return on Zalora as well (applicable for  Returns sent from Customer directly to seller)
  • Users can also reject the Return Order if the user wants to reject the return on Zalora. (applicable for  Returns sent from Customer directly to seller)
  • Also when receiving a return in Ordazzle it will also update the Return Received on Zalora. (applicable for  Returns sent from Customer directly to seller)

Blibli

  • New return orders placed on Blibli will also be received in the Ordazzle.
  • But processing done for the order in Ordazzle will not be updated to Blibli as Blibli does not have any update APIs.

For other Marketplaces (Tokopedia) & Webstores (Magento, Shopify & Custom) Sales return Order functionality is not integrated but users can create manual return for the Delivered Orders of the Channel in Ordazzle. And process the return to arrange return shipment & receive product back at warehouse.
