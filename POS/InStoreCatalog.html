Use Case of In-Store Catalog:

In ETP Unify “ Instore catalog” in a POS system is a fundamental tool for managing products, inventory, and sales transactions in a retail store. It streamlines various aspects of the retail operation, from product management and helps retailers provide a smoother and more efficient shopping experience for their customers.

Capabilities Of In-store Catalog in ETP Unify:

Product Management: The instore catalog helps store owners and employees manage their inventory efficiently. It allows them to add, edit, or remove products from the catalog as needed. This is particularly useful for retailers with a large and frequently changing product selection.
Inventory Control: With an instore catalog integrated into the POS system, inventory levels can be automatically updated as products are sold. This helps prevent overstocking or running out of popular items, which can negatively impact sales and customer satisfaction.
Product Information: Retail employees can access detailed information about each product in the catalog, including product descriptions, images, specifications, and any relevant details. This information is helpful when assisting customers and answering their questions.
Search and Navigation: Instore catalogs often include search and navigation features that make it easy for employees to locate specific products quickly. This is especially important in stores with a vast product range.

Functionality of In-store Catalog:

The User can navigate through ETP Unify→POS→ Instore Catalog option after shift begin.
The User will be able to view 3 Landing Screens they are New Arrivals,Top selling Products,Discounted Products
These screen will populate data based on the tags added in Unify ETP →Store Management→Retail Configuration Template→select Configuration type as Retail Configuration Store Front→ Instore Catalog(Settings)
Note : Any Products shown in this screen will be impacted by Brand store mapping executed for the store.


 


There the user will find 3 multi-select drop downs such as New Arrival Product Tags,Top selling Product Tags,Discounted Product Tags.
Here the User can multiselect tags that are attached to the products in product Master, they want to show case in all the above 3 landing screens(New Arrivals,Top selling Products,Discounted Products)
 


when the mapping is done, the User will be able to view the products attached with specified tags in Retail configuration as per the segments like New Arrivals,Top selling Products,Discounted Products attachment of tags.
The Products details will be displayed with a card view where each card will have a Product Image,The Product Code, The Name of the product and the Offer price of the product available slashed with the Sales price, if no Offer Price then the sales price will be displayed as per below.
 


In the Right handside, the Screen will show the Promotions applicable for the store that the User has logged into. The Promotions will be dynamic based on the promotion Flyers added to any promotions added to the store. Set of 3 promotions will be shown at a time in this space.
The Search Box can be searched based on Product Names, Product Code, Alias number or any product attributes such as Brand, Category or Subcategory.When Clicked on the search, The search will show the Last searched 10 products from the specific Login screen.
The Instore/Endless Aisle Toggle can be used to view the products available in the selected store in the Pos products only, where the Endless Aisle show the Products available across the Company. If the Specific company of the pos selected store has the PDML Flag as Group, then the Products available in the other companies of the same group where the PDML Flag that is the Product date Maintenance level.The details shown in the List- New arrivals, Top selling, Discounted products will only show products and Promotions pertaining to the specific store only.

Hierarchy Details
The Screen will show the hierarchy in the top of the screen, the first level of hierarchy will be shown in the top of the space. when clicked on any of the first level hierarchy, then the screen will show the remaining hierarchy values based on the hierarchy set in Hierarchy Creation.
This is a process to filter the products based on the segments,groups, categories of merchandise to be viewed instantly.
To configure the same, User needs to go back to ETP →Store Management→Retail Configuration Template→select Configuration type as Retail Configuration Store Front→ Instore Catalog(Settings) and the User will find a single selection drop down as “Instore Catalog Hierarchy” (Screenshot as per below) the drop down will show the Hierarchy types created , The User can select appropriate Hierarchy Type and save the template to view the hierarchy details in the Main UI. This activity will show the products filtered as per the selection by the user in the list screen which is after the Landing screen as per below (Screenshot 2)


 

The List Screen
When any product from the Landing screen is clicked or searched from the search box, the screen loads the products and shows in the List screen.
When a Product is clicked from any of the Landing screen, the screen will show all the Products that are attached with the Tags attached under any of the tags mapped to retail config template as per mentioned earlier above in this document.
When search is initiated, then the screen will load all the Products pertaining to the search content and show similar Products in the List screen as per below screenshot shown.The screen will have card views of all the Products Each Card will have Product Image,The Product Brand, The Name of the product and the Offer price of the product available slashed with the Sales price, if no Offer Price then the sales price will be displayed as per below and lastly the Review stars that are available for the Product based on the reviews collected by the specific Product.
Filters
On the Left we will be able to find the Additional attributes that are in correspondence to the products shown in the List. Those additional attributes will be shown based on the Product catalogs additional attributes of the List shown Products based on the search or product selected in the Landing screen when clicked on the plus sign, the User can Multi-select the values available under the selected additional attribute available in the screen.When the User clicks on the see more button, the screen will expand and show all the values available under the additional attribute name from the catalog at the time of creation. The User can multi select and click on the apply button to view the products only pertaining to the selected filter values in the list screen.The filter values selected will be added as tags in the Top row of the screen below hierarchy selections.so that the User will know what the filters applied are.(Screenshots 2 and 3)


Product Attribute Filters
The filters available under additional attribute filters are the product attribute filters, these can be configured in Retail Configuration as mentioned previously, there are 5 attribute drop downs in the retail configuration template screen namely Left side attribute 1,Left side attribute 2,Left side attribute 3,Left side attribute 4 and Left side attribute 5. Where the User can select product attribute values to be showcased on this attribute filter section. The User can select upto 5 attribute filters and the same attribute cannot be reselected between these 5 selections. When the Plus button is clicked under the attribute filter, the values available under selected product attribute will be displayed below and the User can Multi-select the same and view the products pertaining to the filter in the list screen.The filter values selected will be added as tags in the Top row of the screen below hierarchy selections.so that the User will know what the filters applied are.
