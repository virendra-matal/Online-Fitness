Order Processing

The Order Processing functionality allows users to see and manage all orders received into Unify from the configured channels. Users can process orders on this screen, and take actions specific to the order’s status.

Order Processing screen elements- 

The order processing screen consists of 3 sections. 

Order Ticker  

The Order Tickers are tabs available in the list screen to display the orders in the logical groups based on their status. The List consists of 8 tickers - All, Allocation, Ready To Pick, Picked, Packed, Ready to Ship, Shipped, Exceptions, Error. 

The READY TO SHIP ticker displays orders of ‘Ready to Ship’ & ‘Shipment Ready’.

The “SHIPPED” ticker displays orders of ‘Shipped’ and ‘Ship & Invoice’.

The ‘EXCEPTION’ ticker displays orders of status Hold by Rule, Hold by Routing, Rejected, and all In-cancel statuses (as Receive InCancel, RTS InCancel, etc)

On clicking any ticker, the respective order list will populate.

Every ticker also has a count in it. This denotes the no. of orders available in the ticker.

The ‘ERROR’ ticker consists of Error orders that need correction before processing. Further details of Error order processing are given in its section.

Order Actions  

This section consists of action buttons that are required to process the order in Unify. 

Users need to select the order that they need to process using the checkbox available on the left side of order. After selecting an order, action buttons will appear underneath the order tickers, which the user can then accordingly select.

Each button performs a different action on the order. Users can select multiple orders and click the action button to perform actions in bulk. The actions available for bulk processing will depend on the statuses of the selected orders. 

Users can also select individual or partial order lines to process the order partially.

Not all action buttons will be available on the screen. Action buttons will appear based on the status of the selected order, and the level of order selection i.e. single order, multiple orders, or line-level selection

Order List

The order list is where you can find all the orders received from different Seller channels. 

The columns available in the order list are -

  • ORDER# - This is the order number received from the channel. This is always a unique value for a seller channel. Users can also click on this number to open the Summary screen of the order.
  • CHANNEL/SHOP - This is the name of the seller channel defined in Unify from where the order is received.
  • ORDER DATE - This is the date and time at which an order was placed on the marketplace or webstore. The value will be shown in the user's browser timezone.
  • TO SHIP ON - This is the date and time provided by the marketplace before which the order should be shipped in Unify. For Lazada seller logistics, the order needs to be delivered before the TO SHIP ON time to meet the SLA.
  • TIME LEFT - This shows the time left in hours before the TO SHIP ON time. The values do not appear for the orders in the shipped ticker.
  • QTY - This shows the number of products available in the order. Each product can have multiple units ordered in it.
  • AMOUNT - This shows the net total amount of the order. This is the order total without any header charges. This could be different from the total invoice amount. The amount is displayed in the respective channel’s currency format & with the currency symbol.
  • Fulfillment Node - This shows the Node name to which the order is allocated for fulfillment. The values will not be available for non-allocated orders. (Statuses below ‘Allocated’).
  • STATUS - This displays the status of the order. In case there are orders with multiple products & partial processing, then the lowest status of all order lines is displayed here. The different statuses are -

Received - When an order is received in Unify but not allocated to any fulfillment node.

Allocated - When the order is allocated to a fulfillment node where it can be fulfilled.

Ready to Pick (Accepted) - When an allocated order has been accepted by a node.

Rejected - When an allocated order has been rejected by a node. The order will need to be allocated to another node for fulfillment.

Picked - Once a picklist is generated for an order. 

Packed - When the order items are packed at the node

Ready to Ship - When the 3PL is arranged for the order, and the shipping label (Airway Bill) is retrieved from the Logistics partner and printed to attach to the package. An invoice can also be downloaded and attached to the package at this stage.

Manifest Ready - When a manifest is generated and printed for one or multiple orders.

Canceled by Seller - When the seller doesn't want to fulfill the order and decides to cancel the order (at Marketplace or Webstore as well). 

Canceled by Buyer - When the online buyer cancels the order. This can only happen before the order has been shipped.

Delivered - When order status is updated as Delivered from the 3PL in Unify.

Delivery Failed - When the order status is updated as Delivery Failed from the 3PL in Unify

Lost In Transit - When order status is updated as Lost in Transit from the 3PL in Unify

Return To Origin - When the order status is updated from the 3PL in Unify as the delivery was unsuccessful and the package has been returned back to the seller

RTO Received - When the Return to Origin product is received at the seller's node.

Shipped & Invoiced- When order item or order is shipped and invoiced for the order in Unify.

Each order row also has a checkbox on the left side by which the order can be selected for processing.

The right side of the row has the down arrow using which the user can expand the order to see the expanded order view.

Order Expanded View 

When the user clicks on the down arrow in the order list, the order opens in an expanded view.

The expanded view has some header-level information, as well as additional information in 3 tabs once expanded.

Header Level Information -

Tags - The user can see the existing applied Tags to the order. Users can also add or remove any tag using the edit button.

Fulfillment By - This shows if the order will be fulfilled by the Marketplace or Seller

Payment Method - This shows if the order is Prepaid or Cash on Delivery.

Order Rule Check - This shows the status of the Order Rule for this order, if it failed, then it also shows the reason for failure in the ‘i’ icon near it.

Order Allocation Status - This shows the status of Smart Routing for this order, if it failed, then it also shows the reason for failure in the ‘i’ icon near it.

In case the respective rule is not available for the Seller channel then both Order Rule Check and Order Allocation Status will be blank.

Product details -

This section displays line-level information on the order. Users can also select the checkbox provided at every line to process orders partially. The line-level details are -

Product Image - This shows the default image of the product from Unify. If separate images are available for variants, they will be displayed here.

Product Name - The name of the product will be displayed here.

SKU Code - SKU code of the product is displayed here.

Unit - This displays the line quantity of the ordered item.

Stock Info - This displays the total stock available at the node (On Hand Stock) and the remaining stock after excluding the already allocated stock (Net Available Stock).

Pre-allocation: This displays the Transfer Order  (Doc. Number) and Estimated Time of Arrival (ETA).

Picklist No - This shows the number of the picklist if an order is a part of any picklist. 

Priority - Standard / Express priority is displayed here.

Logistic By-Seller / Marketplace logistics is displayed here. Order processing flow changes based on this value.

Delivery Method - BOPIS (Buy Online & Pickup in Store) / Delivery is displayed here. 

Shipping Type - The carrier’s shipping type is Pickup / Dropoff displayed here. 

Unit Price - Base price of the product received in order.

Discount - Total line discount applied on the product. The details are displayed in the ‘i’ icon to show multiple promotions if applied.

Tax & Charges - Total line charges & taxes applied on the product line. The details are displayed in the ‘i’ icon to show multiple charges & taxes if applied.

Net Amount - Displays the net amount of the order line. (amount total for all the line quantities, excluding the line charges). All amounts are in channel currency.

Order Quantity - This shows the total product quantity at the line level.

Shipped: It shows the total quantity shipped for the specific product at the line level.

Balance: This denotes how much quantity of product or order are not shipped.

To Ship/Handover: It Shows the quantity available to ship in-store.

Carrier & Tracking No - This shows the name of the carrier received from the marketplace (for MP logistics order) or selected by the seller (for seller logistics order). Additionally, it displays the tracking number of the order line. It also has an edit icon using which the seller can select or edit the carrier for the order line.

Invoice - This displays the invoice no. of the order or order line

Manifest No - This displays the number of the manifest that the order is part of.

Customer Details -

Customer name, Contact Number, Email ID - Customer’s details as received in order will be displayed in this section

Shipping Address - Shipping address details as provided in the order will be displayed here. The same address will be used to book the shipment for a ‘seller fulfilled’ order. Users can edit this address as per the customer’s request (up to Pack status) using the pencil icon near it.

Billing Address - The customer’s billing address is provided here, if no address has been given, this field will be blank.

After shipping the order, Unify will also display the anonymization date after which customer data will be removed (if anonymization is ON)

Communication -

Comment box - In this section users can enter comments for the order. Users can also tag other users in the comment so that a notification will be sent to the commented user. They can enter ‘@’ to see the list of users to select in the comment.

Comments List - Users can see the previous comments entered by other users in this section. Only the last 5 comments will be displayed here. Other comments can be seen in the summary screen.

Status Progression Bar -

This displays the progression of the order line status. All the past statuses of the order line are displayed here in the sequence.

Along with status it also shows the date and time at which the status was updated for the line.

The statuses of the nodes that had rejected the order are not displayed here.

The summary view can be opened by clicking on the Order#, or by clicking the ‘View’ button. This screen consists of 4 sections.

Order Summary 

This section contains all header-level information of the order. Along with this, there are some additional details the user can can find in this section.

Acceptance deadline - This shows the date and time provided by the marketplace before which order should be accepted as a user. This is currently applicable for Tokopedia only.

Remarks from MP - Any external remark received from the marketplace during order insertion are displayed here.

Order amount Details - This is the entire section at the top right side of the Summary screen. This shows all the line totals as well as different header amounts of the order. The details are as follows -

  • Order items - This is the count of order lines in the order.
  • Product Name, SKU Code & Line Total - This shows total line amounts of every line in the order. (Inclusive of line charges)
  • Taxes & Charges - This part shows the total charges & exclusive taxes applied at the header level. This is followed by the distribution of those header charges, and charge discounts as Charge Name & Amount.
  • Discount - This part shows the total of the discount amount applied at the header. This is again displayed below with the distribution of every promotion name & discount amount.
  • Total Wt - Total weight of order products in grams
  • Total Vol - Total volume of order products in cubic cm
  • Total - This is the total amount of the order (Invoice amount)
  • Payment & Balance Due - This shows paid amount & pending amount of the Order Total

Order Details 

This section displays all the line-level details of the order. Apart from some common details that are available in the product details section, the user can find the following additional details in this section.

Is Gift - Shows if the product is a gift to the receiver.

Gift Message - Any message put in by the customer for the gift product.

Order Package Ref - If the order line has a package ref number (saved during the pack action) then that is displayed here.

Plan Pickup Date & Time - Shows the planned Pickup Date & Time added by the user during the ‘pack’ or ‘carrier selection’ stage. Users can also edit the date & time using a pencil icon.

Dimensions - This displays the Length X, Width X, and Height (in cm) of the product as available in Unify.

Weight - This displays the Weight (in grams) of the product as available in Unify.

NMV - This shows the Net Merchandise Value of the order line as calculated by Unify.

Order Activity Log 

This section displays the list of activities happening in the order. The activities are displayed in order line-wise. 

Every activity record displays the activity message, time & any reason/remark added by the user during the activity.

Communication -

This shows the same section of order communication as in the expanded view of order. But all the previous communication related to this order can be seen here.

Other Functions in the Order List 

Order Filter 

Users can filter the list of orders using the Filter icon available on screen. On clicking the button, it opens the filter panel that has different criteria to apply order filters.

Users can select one or more filters at a time & apply it. Applied filters can be seen in the order list & users can remove any criteria from there as well.

Users can also save a particular set of filter criteria using the ‘Save Filter’ option. This will create a filter view that can be used for personal view or can also be shared with others using the button of ‘Share with others’.

Order Export 

Users can export 

selected orders in the pdf document to get all the order data in tabular format.

If the user wants to export orders of a particular ticker, they can click on Export without selecting any order. The all orders of the current ticker will be considered for export.

Alternatively, the user can also manually select multiple orders and export them

Order Search & Sort 

Orders can be searched using different order fields like Order Number, Tracking No, Order Date, Seller Channel, etc.

Orders can also be sorted based on different criteria like High Net Amount, Low Net Amount, High Qty, Low, Qty, etc.

Refresh 

Users can click on the refresh button

 provided on the screen to get updated records of orders on the list screen

Processing Order in Unify 

In most cases, users can start processing orders from Allocated or Ready to Pick Ticker.

  • In the Allocated Ticker, users can process orders by accepting them -

Select Allocated order, and click on Accept button. The status will be changed to Ready To Pick.

This will accept the order & allocate the stock to the order from the Node inventory.

The order which having Delivery Method as Delivery for that order only needs to select the shipping carrier(Selection of carrier in allocated, and Ready to pick ticker only).

  • In the Ready To Pick Ticker users can process orders by generating picking lists if required, or by packing the order -

Select 1 or more Ready to Pick orders, and click on Generate Picking list button. The status will be changed to

Packed.

This will generate a pick list document for the order which can be downloaded by clicking on Download Picking List Users should also select the carrier for seller-fulfilled logistics before packing it.

  • Select Ready to Pick or Picked order, and click on the Pack button. Users can add required details & save. The status will be changed to Packed.
  • In the Packed Ticker, users can process orders to Get or Generate shipping labels for the order. Details of this process are explained in MP Logistics & Seller Logistics Order Processing.

Select Packed Order & click Get Shipping Label or Generate Shipping Label or Upload Shipping Label

The status will be changed to Ready to Ship.

  • In the Ready To Ship Ticker users can process the order to Get Manifest & Ship. Select Ready to Ship order, and click on Get Manifest. This is an optional step.

Select Ready To Ship order, and click on Ship. On adding data & saving, the status will be changed to Shipped. and while clicking on Ship & Invoice button, the Invoice will generate when payment is completed for the product or order.

It is same for the partial order or product processing.

Additionally, payment can be completed with all payments for pending transactions by navigating to the Payment screen.

The invoice is generated and printed for the same.

In the Shipped Ticker, users can process orders for manual status updates of Delivered, Delivery Failed, Return To Origin & RTO Received (Only for orders with custom carriers in seller logistics order processing).

Select Shippedorder, and click on Mark Delivered. On adding data & saving the status will be changed to Delivered.

Select Shipped order, and click on Mark Delivery Failed. On adding data & saving with the Reattempt option the status will be changed to Delivery Fail.

On saving with the RTO option, the status will be changed to Return to Origin.

Select Shipped or

Delivery fail order, click on Lost in Transit. On adding data & saving the status will be changed to Lost in Transit.

Select Return to Origin order, click on RTO Received. On adding data & saving the status will be changed to RTO Received.

From the time an order is placed up to the "Ship & Invoice", there is the option to make both partial and full payments for the order in the 

Payment action button for above all tickers.

  • The second ticker from the right is the Exception ticker. This consists of orders that need special action before standard processing. It has orders held by Order Receiving Rule(Hold By Routing) and Smart Routing Rule(Hold By Rule), All InCancel status orders for which a buyer’s cancellation request has been received & orders rejected by the fulfillment nodes.

The details about the processing of exception orders are provided here - Link(ORDER EXCEPTION+error)

  • The last ticker in the screen is the Error ticker, this consists of orders that are not yet inserted into Unify because of API or web service mapping exceptions. These orders need to be corrected & then reprocessed.

The details about error order processing are given here Link(ORDER EXCEPTION+error)

  • A Cancel action is available for orders or products within orders prior to reaching the "Shipped" status. This action enables the cancellation of orders and products, along with the possibility of generating refunds in cases where advance payments have been made. This is accomplished through the Tender screen.
  • The status is marked as Seller cancelled.
