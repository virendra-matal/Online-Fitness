Charge Definition

The charge definition allows users to define charges which will be further applicable on billing, transfer order and purchase orders.

Charge in business can be termed as demanding an amount of money as the price for a good or service.

There are multiple types of charges: delivery charge, other charges, GST charge, UTGST, VAT,TCS etc.

Example: Let's say a customer buys a product ABC for price 100$ from Electronics Store on billing customer asks for delivery and store user applies delivery charge of 5$ to the product. 

Therefore now customer will have to pay total 105$ to store and bill will have below line items:

ABC  100$

Delivery charge 5$

Total amount : 105$

List Screen

Over here, users can view all the existing Charges created. Users can edit and delete the records, and also can add new ones.

You can use the search bar to search existing charges by the Charge code or Name on the list screen.  

Each record in list screen has three dots ‘

’,  against it and they will have below following options

  • View - This option will open up the creation screen with pre-populated data for users to view, in read only mode.
  • Edit - This option will open up the creation screen with pre-populated data for users to edit, users can details in this mode.
  • Delete - This option will allow users to delete the record only if it is not used.

Charge Definition creation

You can define a new charge by clicking on the ‘

’  add button available in the main screen on the bottom right corner.

On click of the “+” button , a screen will be visible to create a new charge definition.

Below are Fields where you will be able to define data:

  • Charge Code : The unique code of a charge. 
  • Charge Name : The name of a charge. This name is printed on all external documents
  • Charge Description : Detailed description for defining charge.
  • Charge Status : Status for charge can be active or inactive.
  • Charge Category : The categories for which charge will be defined.
    • Basic Value 
    • Tax
    • Discount
    • Charges
    • Net Total
    • Gross Total
  • Inclusive Charges : This means that if there are any charges other than the item charges, then those charges are termed as Internal Charges.
  • Charge For : The Charge applicability for which charge is defined.
    • Header Charge (If charge is defined for header then it will be applicable charge model defined for header)
    • Line Charge
  • Calculation Method : The calculation factor that is used to calculate the charge amount. 
    • Quantity Based
    • Value based
  • Charge Handling : The type that is used to handle various charges.
    • Flat Value (This means a value will be defined for charge and it will be applied without checking any condition)
    • Condition Based (This means a value will be defined for charge and it will be applied post checking defined condition)
  • Calculation Type : This calculation type is used to calculate Flat value charge handling. A charge is calculated by multiplying the calculation factor and the base selected in the Calculation method field. 
    • Percentage Based (Calculation will be percentage based. “Currency Conversion Factor”, “Currency” and “Exchange Rate Type” will remain Inactive when this option is selected)
    • Amount Value (Calculation will be amount based. “Currency Conversion Factor”, “Currency” and “Exchange Rate Type” will remain Active when this option is selected)
  • Charge Value : This value will be defined only when charge handling is Flat value.
    • The value will be in Percentage when “CalcType = Percentage”
    • The value will be Amount when “CalcType = Amount”
  • Charge Sign : This sign indicates whether charge is positive or negative. This will be used only in case of Header Charges.
    • + (Plus / Positive)
    • – (Minus / Negative)
  • Charge Applicable : The charge applicable indicates that charge is applicable on which type of transactions.
    • Sales
    • Purchase
    • Transfer
  • Print Charge : This indicates whether charge will be printed on documents or invoices.
  • Charge Type : This indicates the type of charge used in charge definition.
    • CGST
    • SGST
    • IGST
    • UTGST
    • Others

 

  • Charge Currency : This indicates the currency for which charge is defined ].(example USD)
  • Exchange Rate Type : This indicates the exchange rate type for which charge is defined. (example: Fixed exchange rate)
  • Charge Definition Condition Details : 

This section will allow defining conditions for condition based charge handling. Based on the below defined fields conditional charge will be applied on transactions.

Example: Users may say that a delivery charge of 5$ will be applicable on invoice having products of Bags, then the same can be done using below fields.

  • Attribute Type (1,2&3) : This field indicates the attribute type for which charge is defined. (It can be selected upto 3 levels, field will have attributes related to product, customer & store) 
  • Attribute Value (1,2&3) : This field indicates the attribute value for which charge is defined. (It can be selected upto 3 levels, field will have attribute values  related to selected attribute type)
  • Valid From Date : This field will define the start date from when the charge slab will be active.
  • Valid To Date : This field will define the end date for the charge slab.
  • Condition Based Handling : This field will indicate whether charge handling will be based on flat value or slab.
    • Charge value (A value will be defined in charge value field if this option is selected)
    • Slab (A charge slab selected will be applicable on transactions if this options is selected)
  • Slab : This field indicates which charge slab will be used for charge calculation while transaction processing.(This will be diabled if ‘Charge Value’ is true for condition based handling)

Slab will ensure that a particular charge Amount will only be applicable when the Line/Header Amount falls within a particular Slab Range.

E.g., Slabs ABC is selected here. (It has the following Slabs

1001 - 2000 = Slab Value 150

2001 - 3000 = Slab Value 250)

Consider that this applicable on Header level only

Now during the billing if the Bill Amount is 1250 then the Charge Amount 150 will be applied directly.

  • Charge Value : This field indicates the charge value defined for above conditions, i.e. this value will be applicable only if defined conditions are true.

(This will be disabled if ‘SLAB’ is true for condition based handling)

  • Discard button - On click of this button users will be able to remove the captured data in above fields, and it will be reset to default view.
  • Add button - This button allows the user to add defined conditions for charge handling.

 

Once a user clicks on add button records will get displayed in the grid and it will have edit and delete option against it if the user wishes to change the existing value then the same can be done using the edit option.

  • Discard button : This button on the creation and edit screen will allow users to redirect to the list screen without saving the changes.
  • Save button : This button on the creation and edit screen will allow users to redirect to the list screen post saving the changes.
