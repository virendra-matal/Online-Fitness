Charge Model

This feature of Charge Model will allow business users to create multiple charge Models that will be valid for a certain time period which can be applied for Bill/Invoices for respective stores. Charge models can be created for both header level and line level charges and can be defined for Flat Values or with condition based values charge definitions.

Example 1 : Zara Tshirt and Shirt Products are applied with CGSTTax , Charge Model can be defined for Header and Line .

Formula for Tax Calculation = Amount* Tax Percentage /Total Tax Amount Applicable.

Rounding off Tax will be displayed.

Example 2 : Zara Tshirt and Shirt Products are applied with CGST and SGST Tax , Charge Model can be defined Line Wise.

Formula for Tax Calculation = Post Discounted Amount * Tax Percentage /Total Tax Amount Applicable.(= 8*5/105  and 35*18/118)

Rounding off Tax will be displayed.

Charge Model can be defined for Sales,Purchase and Transfer Transactions.

To access Charge model master: Login into Unify-> click on Pricing and Promotions in main menu-> Charge Model.

Key Elements For creation of Charge Model:

  1. Base Amount 
  2. Tax Type Applicable
  3. Other Charges (if Applicable)
  4. Net Amount
  5. Discounts (If Applicable)

Charge Model is functional, the system have following options on clicking menu.

  • Add:  System have provision to add new charge Model which will be applicable for respective store billing /Invoice.
  • Edit – System will allow to edit the existing records and values can be edited with new values..  
  • Save: System allow to save the new or edited records
  • View: System will display the existing records on click of search option.
  • Delete - This option will allow you to delete the Charge Model added.

Charge Model Creation

To create a Charge Model click on - Add button- Users have provision to create new charge Model for header level or line level which will attached in the retail configuration template and will be applicable in billing for the stores to which these templates are configured in store configuration group.

Below are the fields where users can define for Charge Model.:

  • Charge Model Code – Unique Identification Code can be defined.
  • Charge Model Name - Unique Identification Name can be defined.
  • Charge Model Description – Description of charge Model to be created.
  • Applied stored - System allow to Active or deactivate the charge model record.
  • From Date - System allow to set from date for respective charge model.
  • To Date – System allow to set to date for respective charge model.
  • Charge Code – System allow to select charge code from drop down list.
  • Sequence No – System set the sequence number on selection of charge code.
  • Charge Sign - System allow to select the charge sign using which the calculation to be considered.
  • Base Charge 1-  System allow to select the base charge (SGST,IGST etc..) from drop down list on which tax calculation can be applied.
  • Formula preview – System will auto populate the formula on selection of charge code, charge sign and base charges.
  • Reset – System will reset all fields to blank.
  • Add – System will add the record and display in list view.

Charge Model List view

List of saved Charge models will be displayed.

  • Charge model Name: This column will display the unique identification Name to which the Charges are assigned.
  • Charge model Code: This column will display the unique identification Code to which the Charges are assigned.
  • From Date: This column will display the from date from which the Charges will be applicable.
  • To Date: This column will display the to date till which the Charges will be applicable.

Charge model Functional Validations

  • System will display validation message on not entering mandatory fields- Following fields are mandatory Charge Code, Seq No, Charge Sign(s) & Base Charge(s)
  • System will display validation if From date and to date overlap with existing charge model.
  • System will display validation if To date is less than from date.
  • System will display validations if From date is less than system current date.
  • System will display validations if store already defined/tagged, system does not allow to edit /delete the record.
  • System will display validation if following are not defined Charge model Name, Charge model Description, Charges For, Charge model Type, From and To dates.
  • System will not display Formula Preview if any of following are not selected Charge code, Sequence No, charge sign, base charge.
