Charge Slab

The charge slab allows users to define charge slabs with different ranges & calculation methods for a particular charge amount so that the charge will be applicable on the invoice line/header value if it falls in the defined range.

For Example: If Slabs ABC is selected in the charge model and it has below defined values

0 - 99 = Slab Value 10

100 - 199 = Slab Value 20

Consider that this charge model is applicable on Header level only

So now during the billing if the Billing Amount is 99 USD then the Charge Amount of 10 USD will be applied directly on the bill.

Impacted Area: This master will have its impact on charge definition, POS billing and purchase orders.

List Screen

Over here, users can view all the existing Charge Slabs created. 

Users can delete and edit the records, and also can add new ones.

The users can sort or use the search bar to search existing slabs by the Slab code or Name on the list screen.  

Each record in list screen has three dots ‘

’,  against it and they will have below following options

  • View - This option will open up the creation screen with pre-populated data for users to view, in read only mode.
  • Edit - This option will open up the creation screen with pre-populated data for users to edit, users can edit names and slab range details in this mode.
  • Delete - This option will allow users to delete the slab record only if it is not used in charge definition.

Charge slab creation

You can create new slab by clicking on the ‘

’  add button available in the main screen on the bottom right corner.

On click of the “+” button , a screen will be visible to create a new charge slab.

Below are Fields where you will be able to define data:

Charge Slab - Header Details

This section allows users to define basic details for a charge slab creation such as code, name, status and calculation method.

  • Charge Slab Code: Users will be able to define the unique code for the charge slab.
  • Charge Slab Name: Users will be able to define the name for the charge slab using this field.
  • Value Calculation : Users can select the calculation method which will be used for further charge calculation.
  • Status : This will be a toggle button to make the charge slab active or inactive.
  • Charge slabs - Range Definition

This section will allow users to create multiple charge slab ranges which can be applied on total bill value or line value. If it is applicable on total bill value then below example can be considered for defining the charge slab.

Example: Users may say that if the total bill value is in the range from 0 to 99 USD will have the charge slab value is 10 USD.

  • From Value - This field will be used to define the From value for the slab range.(example: 0 USD)
  • To Value - This field will be used to define the To value for the slab range. (example: 99 USD)
  • From Date - This field will define the start date from when the charge slab will be active.
  • To Date - This field will define the end date for the charge slab.
  • Slab Charge Value - This field will be used to define the slab charge value for the slab range. (example: 10 USD)
  • Currency - User can select the currency for which he/she is defining the charge slab range.(example USD)
  • Exchange Rate Type - Users can select the exchange rate type for the defined charge slab. (example: Fixed exchange rate)
  • Discard button - On click of this button users will be able to remove the captured data in above fields, and it will be reset to default view.
  • Add button - This button allows the user to add the defined range under SLAB.

 

Once a user clicks on add button records will get displayed in the grid and it will have an edit and delete option against it if the user wishes to change the existing value then the same can be done using the edit option.

  • Discard button : This button on the creation and edit screen will allow users to redirect to the list screen without saving the changes.
  • Save button : This button on the creation and edit screen will allow users to redirect to the list screen post saving the changes.
