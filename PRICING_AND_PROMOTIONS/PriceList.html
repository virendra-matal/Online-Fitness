Price List 

The price list master allows users to manage and create a price list for sales, purchase & distribution transactions. The Price list can be created against customer, store & channel (webstore) attributes. This created price list is executed based on specific to Generic  attributes defined in a price list as price list priorities. 

Example: 

For a product ABC base price is defined as 100 while product creation.

Let's say the user defines the new price list P1 for attribute Customer_Type [Employee] with 10% margin on the base price.

Add a record in the price list with the validity dates 2022-10-01 to 2022-10-31, price 90.00.

Now while billing the product on POS post scanning the product system will check for the most specific price list i.e. if the customer is an employee then the system will apply price list P1 (90) to the product. If the customer is not an employee then the original price of 100 will be applied to the product for billing.

Impact : This price list will be applicable on POS billing execution at stores & channels (webstores) based on the attribute selection.

List Screen

Over here, users can view all the existing price lists created. 

Users can delete and edit the records, and also can add new ones.

The users can sort or use the search bar to search the existing price list by the Price list code or Name on the list screen.  

Each record in list screen has three dots ‘

’,  against it and they will have below following options

  • View - This option will open up the creation screen with pre-populated data only in view only mode with all fields disabled.
  • Edit - This option will open up the creation screen with pre-populated data for users to edit, users can edit price list details in this mode only if price list is not used in billing transactions.
  • Delete - This option will allow users to delete the inactive price list.
  • Copy - This option will allow users to create the copy for the existing price list with all product, customer and other attributes except price list code, name and validity.

Price List creation

You can create new price list by clicking on the ‘

’  add button available in the main screen on the bottom right corner.

On click of the “+” button , a screen will be visible to create a new price list.

Below are the sections required to create a new price list:

  • Price List Basics : 

This section will be used to capture the basic details against the price list such as price list code, name, currency, status, attribute (1,2 & 3)/ values, validity, price list for, promo planning code & rounding off rule.

For example: Users can create a price list as P1 (price list code) Price for Employees (price list name) with currency USD and Active status, against a particular customer type (attribute 1) Employee (attribute value) and store ownership (attribute 2) Own store (attribute value) from 01-12-2022 to 31-12-2022 (validity) for sales (price list for) with rounding of to nearest integer (rounding rule).

This means whenever this price list P1 will be executed it will be applicable only as per above selection made.

  • Sales Price List Details :

This section will be used to filter and add the products in the price list with margin%, price type, tags and validity. Users can also add the products in the price list using the upload function.

Below are the fields used for adding products in price list:

  • Product Attribute1 : Users will be able to select the product attribute in this field against which a price list is to be created such as product category, department, brand etc.
  • Attribute Value1 : Users will be able to select the product attribute value in this field against which a price list is to be created such as for attribute category (t-shirt, shirt values can be selected).
  • Add Attribute : Users can click on this option to add one more attribute row for product selection.
  • Select type : These are radio button options to select users can select any one of the options from SKU, Style & Style-Dimension for which product user wants to create the price list.
  • Dimension : This will be a dropdown list for selection with values as X (size), Y (color) & Z(material). Users can select the dimension of the product for which price list has to be created.
  • Margin % : Users can use this field to define the sales price margin% for the above selected product attribute so that price list records will be created using this margin and roduning off rule. 
  • Price Type : Users can use this field to define the price type for the above selected product attributes.

  • Tags : Users can use this field to define the tags for the above selected product attributes.
  • Valid From Date : Users can use this field to define the start date for sales price against the above selected product attributes.
  • Valid To Date : Users can use this field to define the end date for sales price against the above selected product attributes.
  • Reset : If users do not wish to add the products for above selection made then users can click on this button it will remove the captured data in the fields.
  • Add : To click on this button all above details will be mandatory to fill. If the user wishes to add the products in the price list for above selection made then users can click on this button, on click of this button all products available with the above selection will get added in the price list and the same will be displayed in the grid below.
  • DOWNLOAD TEMPLATE : This button will allow users to download a sample template format for uploading the products in the price list. On click of this button a file will be downloaded in the user's local system.
  • UPLOAD : Using this option users will be able to upload the file for products against which the user wants to create a price list. On click of this button users will get a popup on screen to browse and select the file for upload, if there are no errors in the file then it will be successfully uploaded and the user will get a popup message of successful upload and products will be displayed in the grid. If a file has errors then users will be able to view the errors for each line in the view log function or users can click on view upload history.
  • Price list grid section :

This section will display all products added in the price list with product details (image, product code / dimension, margin%, UOM, tags, Cost, RRP, Sales price, Qty, Price type, validity, Qty slab, last price, Lot ref, Lot no.). Each record in the grid will have an edit ‘

’or delete ‘

’ option against it.

On click of edit button users will get a popup screen for the selected product and they will have the option to edit below fields.

  • Sales price - Users can use this field to change the sales price for individual products.
  • Qty - Users can user this field to change the qty of product (for Eg: Price of say polybags is Rs 5 for 100 qty,  then Qty = 100 and price = 5)
  • UOM : Users can use this field to change the UOM for the selected product, this field will have value only if an alternate UOM or UOM set is defined for the product while creation.
  • Price type : Users can use this field to change the type of price like - promo price for the selected product.
  • Tags - Users can define the tags in this field for the selected product.
  • Validity - Users can use these fields to change the start and end date for the price list against the selected product.
  • Qty Slab - On click of this option users will get popup on screen, here users can define the qty based pricing for the product using below fields, for example: Let’s say product  ABC is having sales price of 10 Rs for 1qty but, now user want to define the qty based pricing for product then user can use this UI and define new sales price for product qty 2 to 10 with unit price as 9.5 Rs.

So now whenever a customer purchases ABC product for 1 qty it will be sold at a price of 10 Rs.

But if the purchase qty is between 2 to 10 then it will be sold at a price of 9.5 Rs.

  • From Qty 
  • To Qty Slab 
  • Unit Price - This field will be used to capture the new sales price for single qty which will be applicable only if the purchase qty is between from and to qty. 

 

 

  • Last price - This is a checkbox which will be used to define that if the defined sales price will be the last price or user will be able to give any further discount. If users click on this check box then while billing this product on POS the cashier will not be able to give any further discount for the product and no further discount from Promotion definition will be applicable.
  • Save : Users can click on this button to save the price list if they want. On click of this button records will be saved and users will be redirected to the main list screen.
  • Discard : Users can click on this button if they do not wish to save the price list. On click of this button records will not be saved and users will be redirected to the main list screen.
