The primary objective of hierarchy setup is to create a well-organized structure that simplifies navigation, categorization, and understanding of data. It allows users to efficiently manage and access information within their specific context. 

 Hierarchy Types and Their Functions: In this system, Hierarchy Groups serve as top-level categories, and each group can have one or more Hierarchy Types, which act as subcategories. 

Hierarchy Creation Master

This master allows users to add criterias for the created hierarchy types against different groups under hierarchy type master. Unlike other masters this master does not have a list screen, data is displayed in tree structure format here.

  • Hierarchy which is available under Etp Unify → Product Management→ Hierarchy Creation when opened will show the hierarchy page.

 

Creating a record -The user needs to go and select any of the hierarchy from the drop-down field to create a new Hierarchy Groups. User can select from the dropdown. Hierarchy Types are like sub category below hierarchy groups, Each group can have one or more hierarchy types.
 

  • Merchandise Hierarchy - Hierarchies belonging to this category will have item number attached to it. Merchandise hierarchy helps to categories products/items/SKUs.

 

Save button flow : 

If user  selected "Merchandise" as the group and a specific hierarchy type (e.g., "Merchandise Base Hierarchy"), follow these steps:

  • In the level 1 list box, create a node (e.g., "MENS WEAR").
  • In the level 2 list box, create sub-nodes (e.g., "CASUAL").
  • In the level 3 list box, create further sub-nodes (e.g., "T-Shirts") ( after selecting each node, in next level list box earlier created nodes will appear by default, so user can either re-use or create new)

 

  • Location  Hierarchy - Location hierarchy helps to categories stores, hence store code will be attached to Location hierarchy

 

Save button flow : 

If you selected "Location" as the group and a specific hierarchy type (e.g., "Retail Location Hierarchy"), follow these steps:

  • In the level 1 list box, choose a node (e.g., "Country-India").
  • In the level 2 list box, create sub-nodes (e.g., "States-Maharashtra").

 

 Time Hierarchy  - Dates will be attached to the Time hierarchy. Time hierarchy helps in doing analysis at different levels like years, quarters, months, weeks and days

Save button flow : 

If you selected "Time" as the group and a specific hierarchy type (e.g., "Season Hierarchy"), follow these steps:

  • In the level 1 list box, choose a node (e.g., "Winter").
  • In the level 2 list box, create sub-nodes (e.g., "Winter SSS1").

 

Employee  Hierarchy  - Employee hierarchy will help to setup complete organization structure, Employee ID from employee master will be attached to the Employee hierarchy.

Save button flow : 

If you selected "Employee" as the group and a specific hierarchy type (e.g., "Item1"), follow these steps:

  • In the level 1 list box, choose a node (e.g., "Department-Retail Operations").
  • In the level 2 list box, create sub-nodes (e.g., "Position-Store Manager, Store Associate, Cashier").
  • In the level 3 list box, further define (e.g., "Grade-G1, G2").

 

Channel Hierarchy  - Channel ID from channel master will be attached to Channel hierarchy. 

 

Save button flow : 

If you selected "Channel" as the group and a specific hierarchy type (e.g., "Integration"), follow these steps:

  • In the level 1 list box, choose a node (e.g., "integration").
  • In the level 2 list box, create sub-nodes (e.g., Marketplace- Country wise ).
  • In the level 3 list box, create sub-nodes (e.g., Shopee, Amazon, Flipkart).

Customer Hierarchy  - Channel ID from channel master will be attached to Channel hierarchy. 

Save button flow : 

If you selected "Employee" as the group and a specific hierarchy type (e.g., "Item1"), follow these steps:

  • In the level 1 list box, choose a node (e.g., "Nationality").
  • In the level 2 list box, create sub-nodes (e.g., "Age").

 

  • Hierarchy Type Name :  This will be a dropdown field for selection users will be able to select the hierarchy type names displayed based on the hierarchy group selected for which they wish to add or delete nodes.

 

  • Users can click on nodes to expand and view rest levels.
  • Users will be able to add new nodes by clicking on the ‘
  • Each record will have three dots,  against it and they will have below following options
  • Edit - This option will open up the hierarchy creation screen with pre-populated data for users to edit, users can edit only hierarchy type names in this mode.
  •  EDIT / VIEW / DELETE - The ETP Unify user can edit, view and delete a record on the Hierarchy creation list screen page by clicking on the 3 dots against the record.

 

Adding and Deleting nodes in hierarchy types

Below are the fields based on which users can add or delete nodes under hierarchy types.

  • Hierarchy Group : This will be a dropdown field for selection users will be able to select the hierarchy group for which they wish to add or delete nodes.

There are a total 5 hierarchy groups:

  • Merchandise
  • Location
  • Time
  • Employee
  • Channel
