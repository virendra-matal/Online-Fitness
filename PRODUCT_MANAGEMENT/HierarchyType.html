Hierarchy Type Master

This master allows users to maintain hierarchy of different groups which is further used for promotion, analytics, ecommerce and product master. 

There are a total 5 hierarchy groups:

  • Merchandise:  This hierarchy type will help users to categorize products based on catalog, brand, department, category, class, sub class and SKU. Users will be able to create only 5 merchandise hierarchies.
  • Location: This hierarchy helps users to categorize stores based on location such as country, state, city, district etc., hence store code will be attached to the Location hierarchy. Users will be able to create only 2 location hierarchies.
  • Time: This hierarchy helps users in doing report analysis at different levels like years, quarters, months, weeks etc. Dates will be attached to the Time hierarchy. Users will be able to create only 2 time hierarchies.
  • Employee: This hierarchy will help users to setup complete organization structure such as departments and sub departments, Employee ID from employee master will be attached to the Employee hierarchy. Users will be able to create only 1 employee hierarchy.
  • Channel: This hierarchy will help users to categorize channels based on web store or marketplace. Channel ID from channel master will be attached to Channel hierarchy. Users will be able to create only 1 Channel hierarchy.
  • Customer: This hierarchy will help users to categorize customers based on customer nationality or Gender.

Note: Users will be able to create Hierarchy Type for the company level only

Hierarchy Type List Screen

  • Hierarchy which is available under Etp Unify → Product Management→ Hierarchy Type Master when opened will show the hierarchy page.
  • Over here, user can view all the existing hierarchy types created for the company.
  • User can sort and edit the hierarchy types, and also can add new ones.
  • User can sort or use the search bar to search existing hierarchies by the Hierarchy group or Hierarchy type Name on the list screen.  

Each record in list screen has three dots ,  against it and they will have below following options

  • View - This option will take you to the hierarchy creation screen with pre-populated data defined during creation in read only mode.
  • Edit - This option will open up the hierarchy creation screen with pre-populated data for users to edit, users can edit only hierarchy type names in this mode.

Creating a record - User  can create new hierarchy type by clicking on the ‘+' add button available in the main screen on the bottom right corner.

On click of the ‘+' add button , a screen will be visible to create a new hierarchy type.

 

 

  • Merchandise hierarchy- Here, users can categorize products based on catalog, brand, department, category, class, sub class and SKU. If the user selects Merchandise Hierarchy then  below options will be displayed. In Merchandise Hierarchy - Base Hierarchy, Hierarchy 1, Hierarchy 2, Hierarchy 3 and Hierarchy 4 can be selected accordingly.
  • Hierarchy Type Name: Users can define hierarchy type names in this field.
  • No. of Levels: Users can select numbers of levels which he/she wants to create for the hierarchy type. On the basis of that Level names will be mentioned
  • Level name: Users can define level names in this field.
  • Auto Create from Attributes: This option allows users to create predefined attribute levels. If the user clicks on the auto create from attributes then User will be able to select attributes on the basis of the hierarchy group selected
  • Promotion: This checkbox allows users to create hierarchy types for promotions. 
  • Analytics (Hierarchy Group Type): This checkbox allows users to create hierarchy types for analytics .
  • E-Commerce(Hierarchy Group Type): This checkbox allows users to create hierarchy types for e-commerce.
  • Level Name(Hierarchy Group Type): This field allows users to add names for levels created in hierarchy. Level name will be given on the basis of the levels selected for the hierarchy group.
  • Save Button : On click of this button the created hierarchy type will get saved and record will be added in the list screen.
  • Discard Button : On click of this button users will be redirected to the list screen without saving the record.
