Product Master Use Cases and Capabilities:

A Product Master in a retail software system is a centralized repository of information about the products a retailer sells. It serves as a critical component for managing and optimizing various retail operations. 

  • Inventory Management: The Product Master helps retailers keep track of their inventory. It stores information about product names, SKUs, descriptions, prices, and quantities. Retailers can use this data to ensure they have the right amount of stock on hand, reducing both overstocking and understocking issues.
  • Pricing Strategy: Retailers can use the Product Master to set and adjust pricing strategies. By storing information about product costs, margins, and competitor pricing, retailers can make data-driven decisions to optimize their pricing.
  • Merchandising: Retailers can use the Product Master to plan and execute their merchandising strategies. This includes organizing products by categories, brands, and attributes, as well as managing product hierarchies and relationships.
  • Order Fulfillment: When customers place orders, the Product Master can help determine product availability, lead times, and shipping options. This ensures that customers receive accurate information about delivery times and product availability.
  • Multi-channel Retailing: In a multi-channel retail environment (e.g., brick-and-mortar stores, e-commerce, mobile apps), the Product Master ensures consistency in product information across all channels. Customers should see the same product details and prices regardless of where they interact with the retailer.
  • Vendor Management: Retailers can use the Product Master to manage relationships with suppliers and vendors. It can store information about vendor contacts, contracts, and performance metrics.
  • Loss Prevention: Retailers can use the Product Master to track and monitor high-value or high-risk items, aiding in loss prevention efforts. This includes tracking serial numbers, security features, and alerting systems for suspicious activities.

Filters

Process of uploading Products

New Upload

Note

When the Template is opened, it will have below sections 

  • Instructions : The Instructions for the Template will be shown here.
  • Multi-select Values : The Common tags will be available for the same the User can use this when creating a product.
  • Product Style : Here as per the product catalog, all the columns will be available. The Mandatory columns will be highlighted with Red and the optional columns will be highlighted as orange.
  • Style : Here the User can add the variant product if available here. For this details to be uploaded, the product configuration in “Products_style column”
  • Hierarchy : Here the User can enter the hierarchy details based on the products we need to upload.

Step One

The User can download a new Template from the “Download Template” screen and the details of the new Products can be entered in this sheet.

Step Two

The User can upload the same sheet in the “New Upload Products” with the Template in one upload space and images in one upload space.

Step Three 

The User will now have to redirect to the “Upload History screen”, here the user can view based on the job id.If some issues are available with the uploaded template the User can download the issue available sheet, then the user can alter and upload the sheet again to add the product.

Step Four

When the details are successfully uploaded, the specific line entry in the upload screen will be shown as completed in the status column.

The issues and status will be displayed in the last 2 columns of the template once upload is completed. Those columns are Upload status - which will mention the upload statuses as Fai, if upload was not completed or Success if the specific line entry is uploaded. If the upload is successful, the specific Record will get uploaded and will be viewed in the product Master List UI.

Marketplace Integration

Over here, marketplace specific information can be input and saved per channel. This includes prices, lead time, planned listing and delisting dates and channel specific SKU codes. Pre-orders for products can also be set up by defining the availability dates.

Product Marketplace Integration Here, User can see the list of Marketplace. Users need to select the Marketplace, based on the selection the Channel shops will be displayed.

Marketplace Category : Users need to attach the MP Category and based on the category the MP attributes will be displayed. User need to fill the attribute details and list the Product on the Marketplace.

Once the user marks the status as Ready to List, the product will be pushed to the channels using Batch Jobs

Offer Name, Offer Price and Offer From Date / To Date : If a User wants to give any offer for a Product he can mention the Offer name and Offer Price. User can also add the offer validity by adding Offer From Date and Time / Offer To Date and Time.

Lead Time To Ship : User can add lead time to ship for Products in Hrs. This will be considered as an SLA for the product and if the Order is received, the Product should be delivered within the mentioned time.

Base Price: User can also attach Sale price Marketplace wise.

Release Date: User can make the Product RTL and schedule the Product releasing date to specific Channel when Product has to be listed.

Delist Date: User can schedule delisting of the Product at specific Date and ETP Unify will delist it on mentioned dateUser can schedule delisting of the Product at specific Date and ETP Unify will delist it on mentioned date.

Availability Date: Availability date can be updated for Pre-order Product

Listing Date: Listing Date is received from the Marketplace once the Product is listed.

Marketplace Attribute: Marketplace Attributes vary from one Marketplace to another and are based on the selected category.

Product Listing Requirements For Marketplace

 

Blibli

  • Image once assigned to the Product cannot be changed
  • Size and color once assigned to the Product cannot be changed
  • Store/Warehouse once assigned to the Product cannot be changed
  • Brand Assigned to the Product cannot be changed
  • New color or Size variant product cannot be added to already existing product.
  • While creating a variant product, the user has to assign all colors and all sizes selected.
  • Offer To and From date in Blibli is currently not supported in the Product creation, but user need to select in ETP Unify by default

Lazada

  • Product item description should not have any HTML tags(such as YouTube link or 3rd party URL link)

Shopee

  • "Offer To" and "From date" are currently not supported in the product creation API of Shopee, if you need to use ``Offer Price with Offer period then use the Promotion Function.
  • Product description should be added in plain text.
  • Video should be MP4 format, max 30 MB size duration should be 10s-60s
  • Product can't be saved with no shipping options selected

Shopify

  • Once the options get created we can not update it
  • You can not delist single item from the style product in shopify, user has to delist all the products

Tokopedia

  • We are removing spaces from SKU Code as it is not allowed in Tokopedia

Zalora

  • In Product creation, size has to be selected for successful listing
  • "Offer To" and "From date" are currently not supported in product creation API
  • New products on sale during product creation are not allowed. Kindly remove the discount given on the Product

Magento

  • For creating a Product on Magento User needs to create his own attributes and attribute set.
  • Additional default attributes with Global Attributes to be created for ETP Unify in Magento by Seller/ProjectTeam
  • All Product attributes from Product catalog will be created as Text Attributes in Magento.Only Style SKU attributes such as color, size will be created as Global and Dropdown type.
