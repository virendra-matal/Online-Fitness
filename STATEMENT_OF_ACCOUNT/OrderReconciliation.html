The Statement of Account function displays financial records of orders and transactions for a particular company. It allows the user to track the transactions that were carried out in a marketplace. The user has the ability to check what amount they have received from the marketplace order-wise, compared to what they were supposed to receive and the difference. There are two sections within the Statement of Account, “Order Reconciliation” and “Payout”.

Order Reconciliation: This section shows the user the transactional record of each order which has been fulfilled

Payout: This section shows the user the total amount that was payable to them in a certain time period

Order Reconciliation:

Order Reconciliation is the function where users can see their transaction details for all the fulfilled orders. The user can use the provided filters to narrow down their search criteria, and click apply to get the results. There is also a field labelled ‘Last Updated’, where the user can see what date and time these records were updated in the system. The user can also choose to search for transactions based on Order Number and Product Code using the search bar.

Filters and Parameters:

Filters:

There are 3 filters that narrow down the search results:

Select Channel Shop: The user can select a single channel for which they would like to see the Order Reconciliation.

Month: The user can select the month for which they would like to view the  data.

Year: The user can select the year for which they want to view the results.

Parameters:

The user will be able to see the order number, product code, shipped quantity, line amount, and shipping fee. In addition to this, commercial fields like commission (charged by the marketplace), transaction fees (a fixed percentage that is charged on the transaction amount, often based on country), other fees (including any other charges which may be applicable), calculated amount (this is the cumulation of the above mentioned fees), payout from MP (this is the calculated amount provided by the MP), difference (difference between the calculated amount, and the payout from MP), and finally the statement number (which is a unique number given to a statement).
