Payout is a summary statement of all the transactions that have happened during a given period of time. Every marketplace has different durations for releasing payout details. A payout statement can be regarding a single order, or multiple orders. This is because the user has 5 filters to narrow down search results. There is also a field labelled ‘Last Updated’, where the user can see what date and time these records were updated in the system.

Filters and Parameters

Filters:

Select Channel Shop: The user can select a single channel for which they would like to see Order Reconciliation for.

Month: The user can select the month for which they would like to view the data.

Year: The user can select the year for which they want to view the results.

Select Status: The user can select the status of the payout amount. The options are ‘Paid’, ‘Unpaid’, and ‘All’.

Statement Number: The user can also input a particular statement number to see that particular statement's payout status.

Common terms used in Payout

Opening Balance: Balance pending from the previous statement. This field is likely to have a value if the previous statement was not paid to the user, or there were any charges pending from the seller side yet to be paid. Based on the two scenarios mentioned, the value for Opening Balance can be either positive or negative.

Closing balance: Closing Balance is the total amount levied on a particular statement. The Closing Balance becomes the Opening Balance in the next statement.

Payout Amount: This is the actual value which has to be paid to the user by the marketplace.
