Payment Modes

The payment modes master allows users to manage and create multiple payment modes. Using this master, different types of payment modes can be created and the same can be mapped to stores by assigning these payment modes to the retail configuration template and that template is attached to the store or store attribute in the store configuration group. These mapped payment modes will be visible on POS while billing.

Example: 

A payment mode ‘Phone Pay’ can be defined by the user for payment type ‘E-Wallets’ within the company user has access.

Now after the payment modes are created, the user can attach these payment modes in the retail configuration template under the retail configuration counter type and payment modes subtype. As these properties are at the counter level user can define what all payment modes will be applicable for each counter within the store.

List Screen

  • On accessing the Payment Modes from the ‘Store Management’ menu a list screen will be shown with existing payment modes created. 

     
  • Users can also sort or use the search bar to search the existing payment modes by the Payment mode name and payment mode code on the list screen.  

Each record in the list screen has three dots 

’,  against it and they will have below following options

  • View - This option will open up the creation screen with pre-populated data only in view-only mode with all fields disabled.
  • Edit - This option will open up the creation screen with pre-populated data for editing, users can edit payment mode details.

New Payment mode creation

Users can create a new payment mode by clicking on the ‘

’  add button available in the main screen on the bottom right corner, a creation page will open up to create a new payment mode.

Below are the sections required to create a new payment mode:

  • Payment Mode Code: The unique code for payment mode.
  • Payment Mode Name: The name of payment. This name is visible on POS while billing and printed on external documents.
  • Payment Type: The payment type for which the payment mode is created can be any type from below.
    1. Cash
    2. Credit Card/Debit Card
    3. Cheque
    4. Demand Draft
    5. Travelers Cheque
    6. Loyalty Points
    7. Gift Voucher
    8. Credit Note
    9. No Refund
    10. Ewallets (Phone Pe, Gpay, Amazon Pay, Paytm, Mobikwik)
    11. Bank Transfers 
    12. Other Payment
    13. Round Off
    14. Foreign Currency
  • Customer Selection: This field identifies whether customer selection will be mandatory or non-mandatory while billing on POS for configured payment mode.
  • Status: This identifies the active or inactive status for payment mode.
  • Payment logo: Users can upload the required logo for the payment mode using this field.
  • Save: Users can click on this button to save the payment mode. On clicking this button, records will be saved and the system will be redirected to the main list screen.
  • Discard: Users can click on this button if they do not wish to save the payment mode. On clicking this button records will not be saved and the user will be redirected to the main list screen.

Editing Payment Modes

  • Considering the user has access to edit rights for the payment modes menu, In the 3 dots edit option will be available for each record. 
  • On clicking on the edit button of a particular payment mode, the edit page will open up where the user can modify the payment mode name, change the status of the payment mode, upload/remove the logo of the payment mode, and enable/disable the customer selection.
