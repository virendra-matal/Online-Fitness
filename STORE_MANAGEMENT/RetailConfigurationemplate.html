Retail Configuration Template

The main purpose of this master is to facilitate the properties configuration centrally. The business use case of this master is to configure properties at the company level which will define the overall functionality usage of the application for the user. Users can create templates of the properties in this master and attach the templates as per the requirement to the stores or a group of stores at attribute level in Store Configuration Group master. This master will be available under the menu ‘Store Management’

For Example : Properties such as ‘Inventory Management Required’ enables inventory management in the application where users will be able to manage their inventory at the store.

The different set of properties are split up under configuration types such as 

  • Retail Configuration Store Operation
  • Retail Configuration Store Front
  • POS Configuration Counter
  • Retail Configuration Invoice
  • Print Configuration Credit Note
  • Lookup
  • Enterprise
  • Retail Configuration Orders
  • Retail Configuration Order Invoicing
  • Retail Configuration Sales Return
  • Retail Configuration Credit Note

LIST SCREEN:

  • On accessing the Retail Configuration Template from the ‘Store Management’ menu, a list screen will be shown with columns and details of  Retail Configuration Template code, Retail Configuration Template Name, Retail Configuration Type, Status.
  • If the user has access to multiple companies within the group, the user can change the company using the bread crumb on the left hand side.
  • On the top right of the list screen, users can search the existing templates created using the search box by entering the template code or template name.
  • Users can also filter the templates by configuration types using the filter dropdown on the top right. There is also an option to filter active or inactive templates.
  • In the list screen for each template there is a 3 dots button which has two options - EDIT and COPY. Given the user has access to these options - they can edit an existing template and modify the properties as per the requirement. 
  • Users can also copy an existing template and create a new one where all the configured properties will be copied from the existing template and users can now modify the required changes and create a new template

Template Creation Page:

  • Users can create a new template using the plus (+) button, which will open up the creation page where users can enter the template code, template name, and select the Retail configuration type and status (by default in active state).
  • Users have to create individual templates for one particular configuration type. A single template cannot be created for multiple configuration types.
  • Retail Configuration Template Code: In this field users can enter up to 5 alphanumeric characters. This is a mandatory field.
  • Retail Configuration Template  Name: This a free field to enter the name of the template to be created. This is a mandatory field.
  • Retail Configuration Type: User to select the type of the configuration required to create the template. On selecting the type relevant properties and subtypes will be displayed where users can start configuring as per the requirement.
  • Status : Users can switch the status to active/inactive.

Retail Configuration Type: Retail Configuration Store Operations:

 

This configuration type consists of subtypes - General Settings, Walk-ins. The properties configured under this configuration type defines the functionality for the functions under Store Management and Store Administration.

 

Retail Configuration Type: Retail Configuration Store Front:

 

This configuration type consists of subtypes - General Settings, Centralized Settings, Customer Settings, Loyalty Settings, Payment Settings, SMS, Email, WhatsApp event templates, Product lookup. The properties configured under this configuration type defines the functionality for the functions under POS menu.

POS Configuration Type: POS Configuration Counter:

 

This configuration type consists of subtypes - General Settings, Peripheral Settings, Mobility Settings, Payment Modes, and Dual Screen Media Upload. The properties configured under this configuration type defines the functionality for the counters of the store for which this template is attached.

Retail Configuration Type: Retail Configuration Lookup:

 

This configuration type can be used to configure fields required in quick product and customer lookup available in the billing screen.

Retail Configuration Type: Retail Configuration Invoice

This configuration type can be used to configure fields required in quick product and customer lookup available in the billing screen.

The users can configure the required invoice formats for billing by selecting the type of invoice, invoice template, logo to be printed on the invoice, address source to be printed on the invoice, and Number of copies to be generated for the invoice.

The users can also define the header text to be printed below the company name and address and also the footer text to be printed at the end of the invoice. The users can also configure the sales persons, charges, and discounts to be printed at the body and footer level of the invoice.
