Return Transfer & Tender Policy-Wiki-ETP Unify

Business Use cases of Returns, Tender and Policy Configuration.

  1. To configure the Returns between stores of the same company based on attributes.
  2. To Assign the Tenders (Gift Voucher, Crédit Note, Loyalty Points) for the returns that are being made for the Return configured stores.
  3. To mention if the returned products should be returned to the originated warehouses or to the specific initially purchased stores.
  4. The above returns can be configured at company Levels in Unified.

To configure Return Returns, Tender and Policy configuration in Unified. 

  • This option will be only available if the All Stores Belonging to Store Attributes is set in →Centralized Sales Return →  Pos Configuration Template → Store Management → Unified.
  • Navigate into Unified → Store Management → Return, Transfer And Tender Policy Configurations.
  • There you will be able to view the List screen if Return Configurations are already available as per below picture.
  • If there are no configurations available then the screen will be blank.
  • If configurations are already available, you will be able to see details of the configurations available.
  • The Columns of the list screen are as below - Receiving store attributes, Billing Store Attribute, Tender Allowed For Redemption, Trigger Back to Origin and a 3 dot button which will have options like - Edit and Delete.
  • The edit opinions will allow you to edit - Tenders Allowed For Redemption  and Trigger Back To Origin options in the configuration screen.
  • The Delete options will delete the specific configuration  from the screen.
  • To  Configure a new Return, tender and Policy configuration, The user has to select a store attribute that is available in the List screen and should click the New configuration button. Once a store attribute is selected, then the same cannot be changed until all the records/Configurations created for the specific store attribute have been Deleted.
  • Then the screen will navigate to the  Return, tender and Policy configuration screen as per below.
  • The first column “Receiving Store Attribute” which is receiving the returned product from the customer will be a Mandatory drop down in which all the store attribute values from common masters for the Store attribute selected in the List screen will be listed down.
  • The User can select any store attribute values from the drop down list.
  • The second column will also be a mandatory drop down, the “Billing Store Attribute” which has initially billed products bought by the customer. This drop down will also have the same values as per above “Receiving Store Attribute”. The user cannot select same values in both the columns
  • Next will be the “Tenders Allowed For Redemption” Here the User can multi-select values like Gift Voucher, Crédit Note and Loyalty Points.  Here User can mention How the return is being Redeemed to the customer. The user can select According to the Business needs on the Tenders that can be allowed for Return.
  • You can then Click on Add and the record will be saved and will be viewed in the List screen.
  • Same configurations cannot be saved twice and same store Attribute values cannot be selected in the same configuration setup in both “Receiving Store Attribute” and “Billing Store Attribute” will be the restrictions which you will not be able to perform in the Returns, Tender and Policy configuration screen.
