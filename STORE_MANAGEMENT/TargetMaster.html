Target Master

Target Master Use Case and Capabilities:

This functionality helps users in assigning targets. Users can use this functionality to assign, view/edit targets not only for employees but also for stores.This functionality will also allow users to assign product wise Target. Using this functionality users must be able to assign targets on a recurring basis.User can assign targets that can have options like Net value, Gross Value (Inclusive of Tax, Inclusive of Discount,))Quantity nos. Etc.

Target Master Screen:-

  • It will be a single screen functionality. At the top right side of the Target Master list screen there will be two filter icons. One is to segregate store and Employee targets with options Store, Employee & All (default) and another is to segregate active & inactive targets with options Active, Inactive & All (default).
  • This screen will showcase existing Active target records created by the user. (Active selected by default in the active/Inactive/All filter). Users can switch to All / inactive using the filter drop-down options.
  • This screen will also consist of a search bar which will allow users to search previously created data;Target code, Store Code, Store Name, Employee Code, Employee Name, From Date, To Date, Target Value  accordingly.
  • View: On click of View, the creation screen will be displayed but all fields will be shown in disable mode except the Discard button.
  • Copy function: The Copy function is available on the three dots where Edit, view is placed. On the click of “Copy” , the selected target will be copied as a new target, but the Employee Code/Name section will be kept blank and the rest of the fields will remain as is. Hence, the User will be able to edit the details already pre populated from the original Target Master.
  • In case of Store target master, the Store, Store Attribute value  should be kept blank, and the rest of the fields will remain as is. Hence, the User will be able to edit the details already pre populated from the original Target Master.
  • If user tries to delete Active target,then user should get a pop up stating that “Active targets cannot be deleted”
  •  User can click on” + “ add button and can select any one of the options for which he wishes to create TargetMaster. After selecting the store or employee, the respective creation form will appear.

Product Attribute Value

User can select multiple product attribute value here and assign a target to them. It will be saved; separated by commas  ( these values will generated from common master).

Target Level

The options will be-Period three dot Recurring While assigning target User can use these methods to assign target.

From Date

Beginning Date of the target. Employees' targets would be counted from this date.

To Date

End Date of the target. Employees have to achieve their target by the end of this date.

Target Type

This field is a dropdown functionality which will fetch list from set up table 

  1. Net Values
  2. Gross Value
  3. Net Quantity

Target Value

This field will accept numeric values and will describe targets to be accomplished. This field will be hidden if the user is selecting the Recurring option in the Target Level field.

Note: The value should be greater than 0

Status

1 - Active

0- Inactive

For Store:

Store Attribute

It's one of the Store group selection fields, the attribute of store will be selected here whose particular values will be selected for filtration. The options will be -

a. Store

b. Store Category

c. Store Subgroup

d. Store Group

e. Store Subtype

f. Store Type

Store Attribute Value

Users can start typing StoreId  or the name of the store and the name will appear as a dropdown (i.e. Lookup should be available). Users can select multiple values in the field.

The purpose of these two fields is to define targets for multiple stores at a time.

Stores Selected

When selecting a store attribute other than ‘Store’, and selecting its values in the store attribute values field (single/multiple), the label should appear below the store attribute value field displaying the count of store coming under selected store attributes.

Hence if Store category is selected & value selected as ‘Jewellery Store’ which is having 14 store under it(as per store master) then the label should appear as Stores

In the Case of Store attribute selected as Company, as there will be no store attribute value selection allowed here, the count will be directly displayed as all the stores coming under the company.

Target Level

The options will be-

Period Recurring While assigning target User can use these methods to assign target.

From Date

Beginning Date of the target. Employees' targets would be counted from this date.

To Date

End Date of the target. Employees have to achieve their target by the end of this date

Target Type

This field is a dropdown functionality which will fetch list from set up table 

  1. Net Values (Considering Discounts  & Charges)
  2. Gross Values (without Discounts  & Charges)
  3. Net Quantity

Target Value

This field will accept numeric values and will describe targets to be accomplished. This field will be hidden if the user is selecting the Recurring option in the Target Level field.

Note: The value should be greater than 0.

Status

1 - Active

0 - Inactive

Edit Option

Users can select the Edit option by selecting it from three dots available on the list screen. When user selects an edit option it will allow user to edit every fields except Employee Code/ Name from employee target master and Store Attribute/ Store Attribute Value in Store Target master. System will not delete any previous records while editing.
