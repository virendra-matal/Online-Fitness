Counter Registration

  • The main purpose of this feature is to have a provision to register counters in a particular store or stores within the company by providing counter registration details, EDC configurations at the counter level, and overriding counter-specific properties (if required) to facilitate the process of retail billing operations in Unify.
  • This feature will also help users to configure EDC devices at each counter level by providing/modifying the specifications like merchant ID, IP address, and port number of each EDC device.
  • This feature will also help users to override the POS properties at a counter level in the counter properties section if required. The POS configuration templates assigned to the store or stores belonging to store attributes will be applicable to all the counters within the store. If any counter specifically requires a change in any property, the users can use this feature to override the properties for a particular counter.

Counter Registration Card View Screen:

  • On accessing the Counter Registration from the ‘Store Administration’ menu, a Card View screen will be shown with the ability to select the store from the drop-down which will populate all the counters available for the selected store.
  • If the user has access to multiple companies within the group, the user can change the company using the breadcrumb on the left-hand side and on changing the company, stores related to the selected company will be displayed.
  • On the top right of the card view screen, users can search the existing counters created using the search box by entering the counter code or counter name.
  • Users can also filter the counters by countertypes using the filter dropdown available. There is also an option to filter active or inactive counters.
  • In the card view screen, for each counter, there is a 3 dots button that has two options - EDIT and EDC Config. Given the user has access to these options - they can edit an existing counter and modify the properties for a specific counter as per the requirement. 

Counter Creation Page:

  • Users can create new counters using the plus (+) button, which will open up the creation page where users can enter the counter code, counter name, and type of the counter and status (by default in active state).
  • Counter Code: In this field, users can enter up to 5 alphanumeric characters. This is a mandatory field.
  • Retail Configuration Template  Name: This is an alphanumeric field to enter the name of the counter to be created Users can enter up to 30 characters. This is a mandatory field.
  • Counter Type: Users have the ability to select the type of counter they want to create. This selection can be made from a dropdown menu, where users can choose from three options: POS, Mobile, or Display.
  • Status: The current status of the counter to be created can be defined using the status toggle.
  • On saving a new counter, a popup message will be displayed indicating that the new counter is created and if the user requires to override the counter level properties for the newly created counter. 

     
  • Users can also configure EDC specifically for the newly created counter by clicking on the ‘EDC config’ button on the top right which will be redirected to the Store EDC configuration menu where users can activate EDC specifically for the created counters.
  • On activating Store EDCs for the counters, activated EDCs will be shown on each card of the counter on the list screen page to indicate the EDCs connected to each counter.

Counter Specific Properties - Overriding properties

 

  • After creating counters, users can override the counter configuration properties by clicking on the counter-specific properties button which will display the template name and template code of the counter configuration template which was attached to the store/store attribute in the store configuration group.

     
  • On overriding properties, the ‘modified’ text will be shown below the property name as an indication that the property was overridden specifically for that particular counter.

Counter Editing:

  • Considering the user has access to edit rights for the counter registration menu, In 3 dots edit option will be available for each card. On clicking on the edit button of a particular counter, the edit page will open up where the user can modify the counter code, counter name, and type of counter required and click on the update button.
  • On clicking update, a popup message will be displayed indicating if the user is required to override the properties specifically for this counter - Counter Updated Successfully, Do You Want To Override Configuration Properties Specifically For This Counter?
  • On clicking Yes, the Counter configuration template which was configured in the store configuration group for the selected store, will be shown along with the properties in the ‘Configuration-Counter’ section.
  • Users can navigate through the available configuration subtypes like General Settings, Peripheral Settings, Mobility Settings, Payment Modes, Dual Screen Media Upload, and override the required properties specifically for the edited counter.
