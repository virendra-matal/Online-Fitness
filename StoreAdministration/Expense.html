Expense

The bank expense list screen enables users to create and manage expenses associated with different cost centers. Users can create an expense record for the store based on the expense type, which is associated with the respective cost centers or the user. Additionally, users can create an expense record, even if it is paid from the head office.

Node Selection: When using the bank expense list screen, users can select the node for which the list screen is to be displayed.

Expense Reporting For Head Office: Users can select the checkbox to filter "expense reporting to head office" the records accordingly when using the bank expense list screen, which is used for reporting the bills received at the store level that need to be processed and paid at the head office.

Three Dots: 

When using the bank expense list screen, users can click on the three dots of any record on the list screen. For records with pending approvals, two options will be displayed, namely "action" and "view". However, for approved records, only the "view" option will be displayed.

Action: Authorized users can click on the "action" option displayed for pending approval records in the bank expense list screen, in order to either approve or reject the expense record that has been created.

View: Users can view the expense records that have already been created for the selected store on the bank expense list screen.

Expense Record Creation: 

When the user clicks the plus button available on the list screen, they can create a new expense record by filling out the necessary information such as the date, amount, category, and description of the expense.

Node: To create an expense record, please select the node for which the record needs to be created.

Expense Cost Center: Users can select the expense cost center from the dropdown menu for which they want to create the expense record.

Expense Type: Users can select the expense type from the dropdown menu for which they want to create the expense record.

Expense Description: Users can enter the expense description in this field.

Date of Expense: Users can select the date of expense i.e. the date on which the bill was generated.

Date of Payment: Users can select the date of payment on which the payee has paid the expense.

Payee Name: Users can enter the payee name i.e. to whom the bill was paid in this field for which the expense record is to be created.

Payment Method: Users can select the payment method (cash or cheque) that was used for the expense.   

Bank Name: If users have selected a cheque as the payment method, please select the bank name from the available options.

Account: Based on the selection of the bank, users can choose the applicable account for the expense.

Cheque No.: Users enter the cheque number in the designated field from which the expense is to be made.

Cheque Date: Users can enter the date of cheque which is entered on the cheque on which the amount of expense is to be deducted.

Receipt No.: Users can enter receipt no. generated after the payment is made for the expense which is made by the store.

Tax No.: Users can enter the Tax No. which is generated on the bill which is generated by the payee.

Amount: Users can enter the amount of expense that is paid by the store.

Remarks: Users can enter the remark if any in this field for the expense done by the store.

Expense Reporting For Head Office: Users can select the checkbox if the amount for the expense to be done is from head office, and this won't affect store cash.

Upload: Users can enter up to a maximum of three images of the bill or expense for which payment is done.
