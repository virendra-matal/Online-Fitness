Retail Configuration Store Specific

The main purpose of this function is to override the properties configuration for a particular store. The business use case of this master is to override the properties at the store level for the templates which were created in the Retail Configuration Template and attached to a particular store or store attributes in the Store Configuration Group. The overriding of properties will only be applicable to the specific store selected for configuration. This master will be available under the menu ‘Store Administration’

For Example: If a property ‘Inventory Management Required’ is enabled in the retail configuration template and if the user requires to disable it for a particular store then the user can use this function to override the property for a particular store.

The properties of the following configuration types can be overridden:

 

  • Retail Configuration Store Operation
  • Retail Configuration Store Front
  • Retail Configuration Invoice
  • Print Configuration Credit Note
  • Lookup
  • Retail Configuration Orders
  • Retail Configuration Order Invoicing
  • Retail Configuration Sales Return
  • Retail Configuration Credit Note

TEMPLATE OVERRIDE SCREEN:

  • On accessing the Retail Configuration Store Specific from the ‘Store Administration’ menu, an override screen will be shown with columns - Store, Retail Configuration Type, Retail Configuration Template Code, Retail Configuration Template Name. 
  • If the user has access to multiple companies within the group, the user can change the company using the breadcrumb on the left-hand side.
  • On selecting the store and the configuration type - the retail configuration template code and template name of the selected configuration type will populate which were attached to the store/store attribute in the store configuration group.
  • The properties which were configured in the retail configuration template will be displayed in the lower panel where the user can now override the required properties for the selected store.
  • Store: The stores belonging to the company selected in the breadcrumb will be displayed in the dropdown.
  • Retail Configuration Type: The user can select the type of configuration required to override the properties. On selecting the type relevant properties and subtypes will be displayed where users can start overriding as per the requirement.
  • Retail Configuration Template Code: The code of the template configured for the store will be displayed.
  • Retail Configuration Template Name : The code of the template configured for the store will be displayed.

Retail Configuration Type: Retail Configuration Store Operations:

 

This configuration type consists of subtypes - General Settings, Walkins. The properties configured under this configuration type defines the functionality for the functions under Store Management and Store Administration.

Retail Configuration Type: Retail Configuration Store Front:

 

This configuration type consists of subtypes - General Settings,Centralized Settings, Customer Settings, Loyalty Settings, Payment Settings, SMS, Email, Whatsapp event templates, Product lookup. The properties configured under this configuration type defines the functionality for the functions under POS menu.

POS Configuration Type: POS Configuration Counter:

 

This configuration type consists of subtypes - General Settings, Peripheral Settings,Mobility Settings, Payment Modes, Dual Screen Media Upload. The properties configured under this configuration type defines the functionality for the counters of the store for which this template is attached.

Retail Configuration Type: Retail Configuration Lookup::

 

This configuration type can be used to configure fields required in quick product and customer lookup available on the billing screen.
