WalkIns

In the context of retail, "walk-ins" refer to customers who visit a retail store without any prior appointments or reservations. They simply walk into the store, browse the merchandise, and make purchases if they find something they like. Walk-ins are an important part of retail business, as they can contribute to the store's foot traffic and sales. Retailers often try to attract walk-in customers through various marketing and advertising strategies to increase their chances of making sales.

The objective of the Walk-ins is to Gather the number of footfalls that have come across the node on a particular day either Hourly wise or for the complete day. The walk-ins or footfalls will also be gathered at the Entry gate ( Maximum 4 ) wise if the node has more than one entry node and if the user has selected the option of creating multiple Entry gates from the POS - configuration Menu. The user can also create Multiple walk-in groups ( Maximum 10 ).

Creating Template - The ETP Unify user can create the template as per the business requirement. The user needs to go to the Retail configuration template,     

The user needs to configure the below details to create walk-ins template -

 

Walkins to Enable - After Clicking on Enable only All the below fields will be functional and entered and checkboxes to be clicked.

No of Days to allow past Walkins to be updated - It's a Numeric Field with + and - options to click. The Maximum numeric Value that can be entered is 365 and the minimum numeric value that can be neutered is 1.

Beginning of the Day Validation - If the user has ticked the checkbox then at the beginning of the day at all the POS for the node, there will be a display of entry mentioning the no of entries missing and the no of days left to fill in the missing entries.

End of the Day of Validation - If the user has ticked the checkbox then at the End of the day at all the POS for the node, there will be a display of entry mentioning the no of entries missing and the no of days left to fill in the missing entries.

Walkins Approval Required - The Walkins entered for the node by the user whether have to be validated for the Approval will be defined by this check box. If the check box is not ticked then at the walk-ins log table it will show the status as Completed.

Walkins entry at hour level - The walk-ins template will provide an entry option at the hourly level if its check box is ticked. The hourly break up of the node will be decided by the node timings which are derived from the fulfillment node setup.

 No Walkin group to be considered to Create Walkins group name - It's a dropdown that will ensure how many walk-ins group names to be considered for the template. It will show a dropdown from 1 to 10. Selecting the no that's selected from the dropdown it will ensure that only those no of walk-in group names are displayed in the template. A minimum of 1 needs to be selected for the walk-ins group template to be active.

Walkin Group Name 1 to Walkins Group Name 10 -   The no of walk-ins that are selected from the dropdown ( no of walk-ins to be considered ) Accordingly those names that have been entered will be visible in the walk-ins template. 

No Entry gates Number to be considered - It's a drop-down field where the user would enter nos from 1 to 4. As per this selection, the number of entry gates selected would determine the no of entry gates to be displayed in the Windows template.

 Entry gate name - The number of entry gates selected would be considered to show the number of entry gate names. Minimum 1 entry gate to maximum 4 entry gates.

User management - The admin user can assign the below functional rights for the walk-ins module to a particular user.

Create Walkins - The User will be able to create a walk-in entry for a particular node under which create action is assigned to him. 

Edit - The user can edit a walk-in entry before the authorization status.

Upload - The user has the right to upload a walk-in file.   

Download - The user has the right to download a walk-ins template CSV file.

Export - The user has the right to export the walk-in data for a Node ( Store ).

Authorize - The user will be able to authorize a walk-in entry under the node to which the function has been assigned.

The user has to go to Store Administration and then click on walk-ins. After clicking on Walk-ins the user will be directed to the Walk-ins list screen. The user can filter records in the list screen by - 

Nodes(Stores) - The ETP Unify user can filter records by selecting the Node ( Store ) from the options.  As the user selects the Store, all the past records of the particular node will be displayed in the grid view.

From Date to Date - The user can filter out records within a particular range of dates.

Statuses - The user can select records having particular statuses like - 

Entry pending 

Approval pending

Completed

Approval

Number of Walk-ins count - The user can enter the walk-in count with a Greater than, lesser than, and equal sign which will filter the records as per the count.

3 Dots - The user can click on 3 dots against a record and then view, edit, enter remarks, and approve a record.

The user can click on + icon or 3 dots and enter details to update walkins count.

As the user will click on enter details then the user will be directed to a new screen - 

The user needs to enter walk-ins count as per walkins group and hourly wise.The user can also enter remarks for the day and also enter count gate wise.

Once the walkins are entered the user needs to save the record by clicking on the save button. When the user clicks on save tab , the user will be directed to the walk-ins list screen with the ecord updated to either Approval pending status or completed status as per the template created.

The user can also create walk-ins record by download and upload process. The user needs to click on 3 dots and click enter details and then click on download. As the user will click on download then a CSV file will be downloaded on the user's system. After that, the user needs to enter the walk-in count, and remarks, and save the file.

After that, the user has to upload this file and click on the save tab. Once the user clicks on save the user will be directed to the walk-ins list screen and the record will be saved.
